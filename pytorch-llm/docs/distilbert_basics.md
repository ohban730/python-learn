# DistilBERT の特徴と仕組み

Hugging Face 社によって開発された BERT の代表的な派生モデル **DistilBERT** の特徴、仕組み（知識蒸留）、およびオリジナル（BERT-base）との違いについて解説します。

---

## 1. DistilBERT とは？

DistilBERT は、2019年に Hugging Face 社によって提案された、**BERTの「軽量・高速化」モデル**です。

モデルが巨大化するディープラーニングの世界において、**「実用的な精度を保ったまま、いかに安価に、高速にモデルを動かすか」**という課題を解決するために作られました。

---

## 2. オリジナル（BERT-base）との性能比較

DistilBERT のパフォーマンスは、オリジナルの `bert-base-uncased` と比較して以下のような特徴を持っています。

* ⭕ **サイズは 40% 削減**: パラメータ数が約1億1000万個から **約6600万個** に減少（約60%のサイズ）。
* ⭕ **速度は 60% 高速**: 処理時間が大幅に短縮され、CPU上でも軽快に動作します。
* ⭕ **精度は 97% を維持**: 性能を極限まで落とすことなく、同等の能力をキープしています。

---

## 3. なぜ精度を保ったまま小さくできるのか？（知識蒸留の仕組み）

DistilBERTの「Distil」は、**知識蒸留（Knowledge Distillation）**という技術に由来しています。

知識蒸留とは、**「非常に賢くて巨大なモデル（教師：Teacher）」の知識を、「コンパクトなモデル（生徒：Student）」に効率よく受け継がせる**技術です。

```
【教師モデル (BERT)】 ─（確率分布を出力）─> [ "apple": 70%, "orange": 20%, "car": 1% ]
                                                    │
                                                    ▼ (この「確率の形」を真似ろ！)
【生徒モデル (DistilBERT)】 ───────────> [ 生徒モデルが同じ確率分布を出せるように学習 ]
```

### なぜこれが効果的なのか？
単に正解ラベル（0 か 1）だけを生徒に学習させるよりも、教師モデルが出力する「曖昧な確率分布（例: `apple` と `orange` は似ているけれど、`car` は全然違う）」という**グラデーションのある知識（ソフトターゲット）**を学習させることで、小さなモデルでも効率よく本質的な意味を理解できるようになります。

---

## 4. アーキテクチャ（構造）の変更点

DistilBERT は、BERT-base と比較して構造上いくつかの要素を削ぎ落としています。

1. **レイヤー（層）数を半分に削減**
   * BERT-base の **12層** のエンコーダーブロックに対し、2層に1層を間引く形で **6層** に削減しています。
   * ※隠れ層の次元数（768次元）などはBERTと同じに保っているため、入力・出力の受け渡しがスムーズに行えます。
2. **トークンタイプ埋め込み層の削除**
   * BERTの事前学習で行われていた「2つの文章の関係を当てるタスク（NSP: 次文予測）」で使用されていた、文章Aと文章Bを区別するための埋め込み層を取り除いています。これにより、さらに軽量化されました。
