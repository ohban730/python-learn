# 有偏分散・ベッセル補正と GPT-2 モデルの互換性

書籍109ページのコラム「有偏分散」に記載されている、統計学的な数理ルールと、AIライブラリ（PyTorch / TensorFlow）の歴史的仕様、および事前学習済みパラメータを再利用するための「互換性」について、詳しく紐解きます。

---

## 1. 統計学の基礎：有偏分散と不偏分散（ベッセル補正）

データ（テンソル）のばらつきを示す「分散」を計算する際、**「データの個数 $n$ で割るか、それとも $n-1$ で割るか」** という統計学上の2つのルールが存在します。

### ① 不偏分散（無偏分散 / Unbiased Variance）
*   **計算式**: データの差の二乗和を **`n - 1`** で割る。
*   **PyTorchの指定**: `unbiased=True` (デフォルト)
*   **ベッセル補正（Bessel's correction）**:
    手元にある一部のサンプル（標本）から、全体の本当のばらつき（母分散）を推定しようとする際、単純にサンプル数 $n$ で割ると、統計的に少し小さめに計算されてしまう（過小評価のバイアスがかかる）ことが数学的に証明されています。
    この過小評価のバイアスを相殺し、本当の分散を正しく推定するために、分母を $n$ から **`n - 1`** に減らす補正を「ベッセル補正」と呼びます。

### ② 有偏分散（Biased Variance）
*   **計算式**: データの差の二乗和を単純にデータ数 **`n`** で割る。
*   **PyTorchの指定**: `unbiased=False`
*   予測や補正のことは考えず、「目の前にあるデータの純粋なばらつき」だけを計算する場合です。

---

## 2. TensorFlow と PyTorch の「デフォルト仕様」のズレ

### ❓ TensorFlow（テンソルフロー）とは？
Googleが開発した、PyTorchと並ぶ世界最大級のディープラーニング（AI開発）ライブラリです。
歴史的に、現在のようにPyTorchが主流になる前（GPT-2が発表された2019年頃）は、TensorFlowが業界の標準ツールでした。OpenAIが開発したオリジナルのGPT-2モデルのソースコードも、このTensorFlowを使って記述されていました。

### ⚠️ ライブラリ間の仕様の違い
実は、AIライブラリの間で、分散を計算するときの初期設定（デフォルト仕様）が異なっています。

*   **TensorFlow の LayerNorm / 分散計算**:
    デフォルトで**「有偏分散（分母を $n$ で割る / `unbiased=False`）」**の仕様になっています。
*   **PyTorch の分散計算 (`torch.var` など)**:
    デフォルトで**「不偏分散（分母を $n-1$ で割る / `unbiased=True`）」**の仕様になっています。

---

## 3. GPT-2モデルとの「互換性（Compatibility）」とは？

書籍の6章で、私たちはOpenAIが何億円もの計算コストをかけて大量のテキストデータから学習させた「本物のGPT-2のパラメータ（重みやバイアスの数値データ）」をインターネット経由でダウンロードし、自分たちのPythonコードに読み込ませて動かします。

### ⚡ なぜ仕様を完全に合わせる必要があるのか？
もし私たちがPyTorchでLayerNormを記述する際、PyTorchの初期設定（分母 $n-1$ の不偏分散）のままにしてしまうと、OpenAIが公式にTensorFlowで学習させた仕様（分母 $n$ の有偏分散）と、**計算方法がごくわずかにズレてしまいます。**

ディープラーニングモデルは、この正規化層（LayerNorm）を何重にも（GPT-2では12〜48層など）通過させて計算を繰り返します。
一見無視できるような「1層あたりの極小の計算ズレ」であっても、何十層も繰り返されるうちに**ズレが指数関数的に増幅され、最終的な出力値が完全に崩壊してしまいます（バタフライ効果）。**

結果として、読み込んだ事前学習済みのパラメータが全く機能しなくなり、デタラメな意味不明のテキストしか生成できなくなります。

> [!IMPORTANT]
> **「事前学習済みの重みとの互換性を確保する」** とは、ダウンロードした本物のパラメータを私たちのPyTorchプログラムに流し込んだときに、公式モデルと寸分違わぬ正しい推論（文章生成）を行わせるために、**正規化の計算アルゴリズムの分母を公式の『有偏分散（分母 $n$）』に完全に合わせる**、という意味です。

## 4. 埋め込み次元 $n$ が大きいときの影響

コラムには「LLMでは、埋め込み次元 $n$ が非常に大きいため、$n$ と $n-1$ の差は実質的に無視できます」とあります。

*   GPT-2の最小モデルの埋め込み次元は $n = 768$ です。
*   仮に差の二乗和が $1000$ だった場合：
    *   分母が $n$ (768) の場合: $1000 / 768 \approx 1.30208$
    *   分母が $n-1$ (767) の場合: $1000 / 767 \approx 1.30378$
    *   その差はわずか **`0.0017`** 程度です。

このように、数値そのものの差は極めて小さく、学習時の統計的なばらつきとしては完全に無視できるレベルです。しかし、「すでに決定された事前学習パラメータを正しく読み込んで互換性を保つ」という点においては、この `0.0017` のズレを排除して計算を完全一致させることが決定的に重要になります。
