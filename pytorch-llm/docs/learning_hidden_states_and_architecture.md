# 「隠れ状態を学習する」の意味とモデル内部構造（アテンション・全結合層・隠れ状態）の整理

書籍の記述にある「隠れ状態を学習する」の正確な意味と、Transformer内部における **アテンション層・全結合層・隠れ状態** の役割と関係性について整理して解説します。

---

## 1. 「隠れ状態を学習する」の真の意味

数学的・厳密に言うと、**「隠れ状態というデータそのものを直接更新する」わけではありません。**

* **隠れ状態 (Hidden State)**:
  入力テキストがモデル内部を通過する過程で生成される**「一時的な中間データ（数値ベクトル）」** です。それ自体は学習可能なパラメータ（重み）ではありません。
* **「隠れ状態を学習する」の実際の意味**:
  **「分類タスクに適した隠れ状態が生成されるように、その隠れ状態を作り出している手前のアテンション層や全結合層のパラメータ（重み $W$）を学習（更新）する」** という意味の短縮表現です。

---

## 2. 隠れ状態・アテンション層・全結合層の関係（全体像）

Transformerモデル（DistilBERTなど）の内部では、入力データが以下のように各層を通過しながら、隠れ状態のデータ表現が順番にアップデートされていきます。

```text
 ［ 入力テキスト ］ (例: "I feel great")
       │
       ▼ 【1. 埋め込み層 (Embedding Layer)】
 ［ 初期隠れ状態 h_0 ］ (単語ごとの辞書的な意味ベクトル)
       │
 ┌─────┴─────────────────────────────────────────┐
 │ Transformerブロック (第1層 〜 第N層)          │
 │                                               │
 │  ① Self-Attention層 (自己注意機構)            │
 │     ・「どの単語がどの単語に注目すべきか」計算  │
 │     ・文脈情報を取り込んで隠れ状態を更新       │
 │       │                                       │
 │       ▼                                       │
 │  ② 全結合層 (Feed-Forward Network / FFN)       │
 │     ・抽出した文脈表現をさらに複雑に非線形変換 │
 │     ・モデルの記憶・知識と照らし合わせて調整   │
 └─────┬─────────────────────────────────────────┘
       │
       ▼ (各層を通過するごとに隠れ状態が深まる)
 ［ 最後の隠れ状態 (last_hidden_state) ］
       │
       ▼ 【2. 分類ヘッド (nn.Linear 全結合層)】
 ［ 最終スコア (Logits) ］ ──> 【3. Softmax】 ──> 確率 (例: 喜び 85%)
```

---

## 3. 各コンポーネントの役割と整理

| コンポーネント | 種類 | 役割と特徴 |
| :--- | :--- | :--- |
| **隠れ状態 (Hidden State)** | **データ (テンソル)** | 入力テキストがモデル内を通過するときの中間表現（ベクトル）。層を経るごとに「単純な単語の意味」から「高度な文脈・感情の意味」へと深化する。 |
| **Self-Attention 層** | **構造 (パラメータ保持)** | 文中の単語同士の関係性（「feel」と「great」のつながり等）を計算し、文脈を捉えた隠れ状態を作る役割。 |
| **全結合層 (FFN)** | **構造 (パラメータ保持)** | Attention層が拾った文脈表現を受け取り、非線形変換を行って表現力を高める役割。 |
| **分類ヘッド (Classification Head)** | **構造 (パラメータ保持)** | 最後の隠れ状態（`[CLS]` ベクトル）を入力として受け取り、最終的な分類クラス（6つの感情）のスコアに変換する出力層（`nn.Linear`）。 |

---

## 4. ファインチューニング時に起きていること

ファインチューニングを実行した際、モデル内部では以下のループが回ります。

1. **順伝播 (Forward)**:
   入力テキストが「アテンション層」と「全結合層」を通過して「最後の隠れ状態」が作られ、分類ヘッドから感情の予測値が出力される。
2. **損失計算 (Loss)**:
   予測値と正解ラベルを比較し、誤差（Loss）を計算する。
3. **逆伝播 (Backward / バックプロパゲーション)**:
   誤差の情報（勾配）が、**分類ヘッド ──> 最後の隠れ状態 ──> 各層の全結合層・アテンション層** へと逆方向に伝わる。
4. **パラメータ更新**:
   勾配に基づいて、アテンション層や全結合層の**重み $W$ が微調整**される。

この結果、次回の計算では**「感情分類がより容易に行えるような優れた隠れ状態」** が各層から生成されるようになります。これが「隠れ状態を学習する」というプロセスの全貌です。
