# データリーク（Data Leakage）とサンプリングの順番

書籍の30ページにある「学習と評価の分割を行う前にサンプリング手法を適用しないように注意してください。さもないと、データリークが発生してしまいます。」というアドバイスの理由と、機械学習の最重要概念の一つである **データリーク** について解説します。

---

## 1. データリーク（Data Leakage / 情報漏洩）とは？

データリークとは、モデルの学習（訓練）時に、**「本来は本番（評価）時に見えてはいけないはずの情報」が、学習用データに紛れ込んでしまう（漏れ出てしまう）現象**です。

テストに例えるなら、**「テストを行う前に、先生が生徒に試験問題の一部を教えてしまった（カンニングできる状態）」**のようなものです。

### データリークが起きるとどうなる？
* **学習中（開発時）**: 「精度99%！完璧なモデルができた！」と大喜びする。
* **本番運用時**: 全く未知のデータに対して予測させると、**「精度が著しく低下し、全く役に立たない」**という状態になります。
* （※評価データがあらかじめ学習データに漏れていたため、開発中の評価テストは簡単に満点が取れただけだった、というオチになります）

---

## 2. なぜ「分割前」にサンプリングするとデータリークになるのか？

ここでいうサンプリングとは、主にデータ数が少ないレアなクラスを複製して増やす **「オーバーサンプリング」** を指します。

```
【 悪い例：分割前にオーバーサンプリングした場合 】

   [ レアデータ A ] ── (複製) ──> [ A_copy1, A_copy2 ]  (同じ中身のコピーが2つできる)
                                        │
                         ┌──────────────┴──────────────┐ (ここで分割)
                         ▼                             ▼
                  【 学習データ 】              【 評価データ 】
                     [ A_copy1 ]                   [ A_copy2 ]

  * 結果: 評価データに入っている「A_copy2」は、学習データにある「A_copy1」と完全同一のデータです。
  * モデルは学習時に「A_copy1」を丸暗記しているため、評価時に「A_copy2」を簡単に見破って正解してしまいます。
  * これが「データリーク」です。モデルの本当の実力（未知のデータへの対応力）が測れなくなります。
```

---

## 3. 正しい手順

データリークを防ぐための鉄則は、**「評価（テスト）データは、モデルにとって完全に未知の存在として隔離しておく」**ことです。

```
【 正しい例：先に分割し、学習データのみをサンプリングする 】

                          [ 生のデータセット ]
                                   │
                    ┌──────────────┴──────────────┐ (★まず最初に分割)
                    ▼                             ▼
             【 学習データ 】              【 評価（テスト）データ 】
              (不均衡な状態)                 (不均衡な状態のまま隔離)
                    │
                    ▼ (★学習データ内だけでサンプリングを実行)
           [ サンプリング後の学習データ ]
                    │
                    ▼ (学習)
              [ 学習済みモデル ] ── (テスト) ──> 【 評価データ 】
```

### 正しい手順のステップ
1. **まず最初に、データを「学習用」と「評価用」に分割する。**
2. 評価用データは金庫にしまうように、一切加工せず保管する。
3. **学習用データの中だけで**、少ないデータをコピーして増やす（オーバーサンプリング）などの処理を行う。
4. 学習を行い、最後に一度だけ、隔離していた評価用データを使って精度をテストする。

この手順を守ることで、評価データの情報が学習に混入する「データリーク」を完全に防ぎ、本番でもしっかり動くモデルを作ることができます。
