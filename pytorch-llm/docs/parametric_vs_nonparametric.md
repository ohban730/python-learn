# パラメトリック (Parametric) と ノンパラメトリック (Non-parametric) の違い

機械学習や統計学における **パラメトリック** と **ノンパラメトリック** という言葉の意味、およびヒューリスティックとの分類軸の違いを整理して解説します。

---

## 1. 結論：はい、まったく「別の切り口（分類軸）」の概念です

* **ヒューリスティック vs アルゴリズム (解き方の厳密さの軸)**
  * 解き方が「直感・経験則の単純ルール」か「数学的な最適化アルゴリズム」か。
* **パラメトリック vs ノンパラメトリック (モデルの構造・パラメータ固定度の軸)**
  * モデルの形が「あらかじめ決まった数式（固定個のパラメータ）」か「データに応じて柔軟に形を変えるか」。

---

## 2. パラメトリック (Parametric) とは？

**あらかじめ「データの分布」や「モデルの形（数式）」を仮定し、決められた一定の個数のパラメータ（重み $w$ やバイアス $b$）を学習する手法**です。

* **イメージ**: 「このデータはきっと一次関数 $y = ax + b$ やシグモイド関数に沿っているはずだ」と決めておき、その数式の係数（$a$ や $b$）だけを学習します。
* **代表例**:
  * **ロジスティック回帰**（重み $w$ とバイアス $b$ の数がデータ件数に関わらず固定）
  * **正規分布を前提とした統計分析 (t検定など)**
* **メリット**: 学習が終わればパラメータ（数値）だけを保持すればよいため、メモリが超省スペースで推論が速い。
* **デメリット**: データが仮定した数式の形から大きく外れていると、精度が出ない。

---

## 3. ノンパラメトリック (Non-parametric) とは？

**「パラメータ（パラメータ数）がゼロ」という意味ではなく、「特定の数式や分布（正規分布など）をあらかじめ固定・仮定せず、データの複雑さに合わせてモデルの形が柔軟に変化する手法」** です。

* **イメージ**: 「どんなに歪んだ形・複雑なデータの形をしていても、データそのものの散らばりに合わせてしなやかにフィットさせよう」という手法。
* **代表例**:
  * **ランダムフォレスト / 決定木**（データの複雑さに合わせて、木の深さや分岐ノードの数が自由に変形する）
  * **k近傍法 (k-NN)**（学習データそのものを保持し、新しいデータの近くにある $k$ 個のデータで多数決する）
* **メリット**: データがどんなに歪んで複雑に入り組んでいても、柔軟に分類境界を作れる。
* **デメリット**: データ量が増えるとモデル自体のサイズが大きくなり、メモリや計算が重くなりやすい。

---

## 4. これまでの学習内容とのマップ

分類軸を整理すると、これまでに登場した手法は以下のように綺麗にマッピングされます。

| 分類 | パラメトリック (数式固定) | ノンパラメトリック (構造柔軟) |
| :--- | :--- | :--- |
| **厳密な数理アルゴリズム** | **ロジスティック回帰**<br>(重み $w$ の個数が固定) | **ランダムフォレスト**<br>(決定木の分岐数が可変) |
| **ヒューリスティック (経験則)** | ー | **DummyClassifier**<br>(「一番多い感情を答える」などの直感ルール) |
