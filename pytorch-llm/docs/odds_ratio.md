# 確率、オッズ、オッズ比、そしてロジットからシグモイドへの数学的ストーリー

ディープラーニングやLLMのコードに頻出する「ロジット（Logits）」という言葉の背後にある、統計学の「オッズ」から機械学習の「シグモイド（Sigmoid）」へと繋がる美しい数学的ストーリーを解説します。

---

## 1. 確率 (Probability) と オッズ (Odds) の違い

日常会話では「オッズ」は単なる確率や倍率として使われがちですが、統計学では明確な数学的区別があります。

### ① 確率 ($p$)
*   **定義**: 全ての結果のうち、ある事象が起こる割合。
*   **値の範囲**: $0 \le p \le 1$ ($0\%$ 〜 $100\%$)
*   *例*: コインを投げて表が出る確率は $p = 0.5$。

### ② オッズ (Odds)
*   **定義**: **「ある事象が起こる確率」と「起こらない確率」の比率**。
*   **数式**:
    $$\text{Odds} = \frac{p}{1-p}$$
*   **値の範囲**: $0 \le \text{Odds} < +\infty$
*   *例*: 表が出る確率が $p = 0.5$ のとき、裏が出る確率は $1-p = 0.5$。
    $$\text{Odds} = \frac{0.5}{0.5} = 1.0$$
    オッズが `1.0` であるとは、「起こる見込みと起こらない見込みが全く同じ（等倍）」であることを意味します。

---

## 2. オッズ比 (Odds Ratio) とは何か？

オッズ比（Odds Ratio / OR）とは、**「2つの異なるグループ間におけるオッズの比率」**です。主に医学や社会科学で「ある要因（例：薬の服用）が、結果（例：病気の回復）にどれだけ影響を与えるか」を測るために使用されます。

### 💊 臨床試験の具体例

新薬の効果を確かめるため、患者を2つのグループに分けてテストしました。

*   **グループA (薬を服用した人)**: 病気が治る確率 $p_A = 0.8$ ($80\%$)
*   **グループB (偽薬を服用した人)**: 病気が治る確率 $p_B = 0.5$ ($50\%$)

それぞれのオッズを計算します。
*   **グループAの治るオッズ**:
    $$\text{Odds}_A = \frac{0.8}{1-0.8} = \frac{0.8}{0.2} = 4.0$$
    （治る見込みは治らない見込みの 4 倍である）
*   **グループBの治るオッズ**:
    $$\text{Odds}_B = \frac{0.5}{1-0.5} = \frac{0.5}{0.5} = 1.0$$
    （治る見込みは等倍）

この2つのオッズの比率を取ったものが **オッズ比 (Odds Ratio)** です。

$$\text{Odds Ratio} = \frac{\text{Odds}_A}{\text{Odds}_B} = \frac{4.0}{1.0} = 4.0$$

#### 💡 オッズ比の解釈
オッズ比が `4.0` ということは、**「新薬を服用したグループは、服用しなかったグループと比べて、病気が治るオッズが 4 倍高い（＝極めて有効である）」**と解釈できます。

## 3. なぜ対数を取るのか？ (対数オッズ / ロジットの誕生)

オッズ（ $0$ から $+\infty$ ）には、数学的に不便な**非対称性**という問題があります。

*   確率が $0.5$（五分五分）のとき、オッズは $1.0$。
*   確率が $0.5$ より大きくなると、オッズは $1.0$ から **$+\infty$（無限大）**まで無限に広がります。
*   確率が $0.5$ より小さくなると、オッズは $0.0$ から $1.0$ の**狭い範囲**に押し込められます。

この非対称性を解決するために、オッズの**自然対数（ $\log$ ）**を取ります。これが **対数オッズ（Log-Odds）**、すなわち **ロジット（Logit）** です。

$$\text{logit}(p) = \log\left(\frac{p}{1-p}\right)$$

### 📊 対数を取ることで起こる魔法（全実数への対称なマッピング）
*   確率が五分五分（ $p = 0.5$ ）のとき:
    $$\text{logit}(0.5) = \log(1.0) = 0$$
*   確率が $1.0$ に限りなく近づくとき（ $p \to 1$ ）:
    オッズは $+\infty$ になり、対数を取っても **$+\infty$** に向かいます。
*   確率が $0.0$ に限りなく近づくとき（ $p \to 0$ ）:
    オッズは $0.0$ になり、対数を取ると $\log(0) \to$ **$-\infty$** に向かいます。

これによって、ロジットは **「$-\infty$ から $+\infty$ までのすべての実数」** を、五分五分（ $0$ ）を中心として左右対称にきれいにカバーするようになります。

## 4. 逆関数としての「シグモイド関数」の導出

機械学習モデル（ニューラルネットワーク）は、計算の過程で制約のない任意の数値（ $-\infty$ 〜 $+\infty$ ）を出力します。これがモデルの出力する生のスコア（**ロジット** $x$）です。

このロジット $x$ を、元の美しい「確率 $p$ （ $0$ から $1$ の範囲）」へと逆変換するためには、**ロジット関数の逆関数（インバース）**を計算する必要があります。

数学的に解いてみましょう。

$$x = \log\left(\frac{p}{1-p}\right)$$

この式を $p$ について解きます。

両辺のネイピア数 $e$ の指数を取ります（対数 $\log$ を外す）。

$$e^x = \frac{p}{1-p}$$

両辺に $(1-p)$ を掛けます。

$$e^x(1-p) = p$$
$$e^x - p e^x = p$$

$p$ を含む項を右辺にまとめます。

$$e^x = p + p e^x$$
$$e^x = p(1 + e^x)$$

$p$ について解くため、両辺を $(1 + e^x)$ で割ります。

$$p = \frac{e^x}{1 + e^x}$$

分母と分子を $e^x$ で割ります（分子を 1 にするため）。

$$p = \frac{1}{\frac{1}{e^x} + 1}$$
$$p = \frac{1}{1 + e^{-x}}$$

#### 🎉 シグモイド関数の誕生！
ロジット関数の逆関数を解いた結果、現れたのはまさに **シグモイド関数（Sigmoid）** です！

$$\text{Sigmoid}(x) = \frac{1}{1 + e^{-x}}$$

*   モデルが叩き出した任意の生スコア（**ロジット** $x$）をシグモイド関数に通すと、数学的な裏付けをもって、綺麗に $0$ から $1$ の範囲の**確率 $p$** へと戻すことができます。
*   この多クラス（マルチクラス）版が **ソフトマックス（Softmax）関数** です。

これが、ディープラーニングにおいて「ロジット」を出力し、「シグモイド/ソフトマックス」で確率に変換するアーキテクチャの根底にある、美しい数理的デザインです。
