# Hugging Face Datasets `map` メソッドの `batched` と `batch_size` パラメータ

書籍の40ページでデータセットのトークン化に使用されている以下のコードのパラメータ `batched=True` と `batch_size=None` の意味と役割について解説します。

```python
emotions_encoded = emotions.map(tokenize, batched=True, batch_size=None)
```

---

## 1. `batched=True` の役割：一括処理モードの有効化

`map()` メソッドは、データセットのすべての行に対して指定した関数（ここでは `tokenize`）を適用する処理です。

* **`batched=False` (デフォルト)**
  * **挙動**: データを1行ずつ（1つの文章ずつ）取り出して処理します。
  * **処理されるデータ**: `{"text": "I feel happy"}` （1個の文字列）
  * **問題点**: 処理速度が非常に遅くなり、文章同士の長さを揃えるパディングも行えません。
* **`batched=True`**
  * **挙動**: データを複数行まとめて（バッチにして）一括で処理します。
  * **処理されるデータ**: `{"text": ["I feel happy", "I am sad", "Good morning"]}` （文字列のリスト）
  * **メリット**: トークナイザーの並列処理（Rustバックエンド）が走り、超高速になります。また、バッチ内の最大長に合わせたパディングが可能になります。

---

## 2. `batch_size=None` の役割：データ全体を1つのバッチにする

`batched=True` にしたとき、「何件ずつまとめて処理するか」を制御するのが `batch_size` パラメータです。

* **`batch_size=1000` (デフォルト)**
  * 1,000件ずつに小分けして処理します。メモリを節約できますが、パディングされる長さは**1000件のブロックごとにバラバラ**（バッチAは最大長40、バッチBは最大長50など）になります。
* **`batch_size=None`**
  * データセットを小分けにせず、**「データセット全体をまるごと1つの巨大なバッチ」として一挙に処理します。**

---

## 3. なぜ今回は `batch_size=None` なのか？（グローバルパディング）

書籍のこの部分では、トークン化したデータ全体をシンプルな行列（テンソル）としてそのまま次の分析や抽出処理に回したいため、**「データセット内のすべての文章を、完全に同じ長さ（Shape）に揃えたい」**という目的があります。

```text
【 batch_size=1000 (デフォルト) 】
  ・バッチ1 (1〜1000行目)の最大長は 40 ──> すべて長さ 40 に揃う
  ・バッチ2 (1001〜2000行目)の最大長は 50 ──> すべて長さ 50 に揃う
  ※ バッチごとに結果のサイズ（次元）がズレてしまいます。

【 batch_size=None 】
  ・データセット全体（例: 16000行）の全文章の中での最大長（例: 65）を調べる。
  ・16000行のすべての文章を、一律で「長さ 65」にパディングして揃える。
  ※ すべてのデータが完全に同じサイズになり、扱いが簡単になります。
```

このように、データセット全体の形状を均一に揃える処理を **グローバルパディング (Global Padding)** と呼びます。これを実現するために `batch_size=None` を設定しています。

---

## 4. トレードオフ（注意点）

| 設定 | メリット | デメリット |
| :--- | :--- | :--- |
| **`batch_size=None`** | 全データが完全に同じサイズになり、一括で行列変換がしやすくなる。 | データ全体を一度にメモリ（RAM）にロードするため、**数百万件規模の巨大データセットで実行すると、メモリ不足（OOM）でクラッシュする。** |
| **`batch_size=1000` (など)** | メモリ消費量が一定に抑えられるため、巨大データでも安全に処理できる。 | バッチごとに次元数が異なるため、学習時には「データコレーター (Data Collator)」などを用いて動的にバッチサイズを揃える工夫が必要。 |

※ 書籍で使用している感情データセット（`emotions`）は数万件規模の非常に軽量なデータセットであるため、`batch_size=None` にして全体を一気に処理してもメモリ上まったく問題ありません。
