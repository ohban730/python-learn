# Hugging Face: pipeline("summarization") で使用可能な主要な引数

Hugging Face Transformers の要約パイプライン（`pipeline("summarization")`）を実行する際、単にテキストを渡すだけでなく、様々な引数を与えることで要約の長さやアルゴリズム、文章の重複をコントロールすることができます。

これらは内部でテキスト生成（Text Generation）を行うためのパラメータに基づいています。

---

## 1. 長さを制御する引数

生成される要約の長さを制限するための最も基本的なパラメータです。

| 引数 | 型 | 説明 |
| :--- | :--- | :--- |
| **`max_length`** | `int` | 生成される要約の最大トークン数。 |
| **`min_length`** | `int` | 生成される要約の最小トークン数（これより短くならなくなります）。 |
| **`max_new_tokens`** | `int` | **（推奨）** 入力サイズとは無関係に、新しく生成する最大トークン数を指定します。 |
| **`min_new_tokens`** | `int` | **（推奨）** 新しく生成する最小トークン数を指定します。 |

> [!TIP]
> `max_length` は入力テキストのトークン数も含んでしまうモデルがあるため、最近のHugging Faceでは純粋に生成する文字数だけを縛れる **`max_new_tokens` / `min_new_tokens`** の使用が推奨されています。

---

## 2. 生成アルゴリズム（探索方法）を制御する引数

どのように次の単語を選んで文章を構築するかを指定します。要約タスクでは、論理的で簡潔な文章を出力させるためにビーム探索（Beam Search）がよく使われます。

| 引数 | 型 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| **`num_beams`** | `int` | `1` | ビーム探索（複数候補を同時に探索する手法）のビーム数。<br>`4` などを指定すると、より高品質で論理的な要約になります。 |
| **`do_sample`** | `bool` | `False` | `True` にすると確率的なサンプリング（ランダム性のある生成）を行います。<br>要約タスクでは嘘を減らすため `False`（決定論的）が基本です。 |
| **`temperature`** | `float` | `1.0` | 生成テキストの「ランダム度（創造性）」。<br>低いほど論理的（決定論的）に、高いほど多様（ランダム）になります。 |
| **`top_k`** | `int` | `50` | サンプリング時に、上位 `top_k` 個の候補単語からのみ選択します。 |
| **`top_p`** | `float` | `1.0` | サンプリング時に、累積確率が `top_p` に達するまでの上位単語からのみ選択します。 |

---

## 3. 重複（ループ）を回避する引数

同じフレーズや単語を何度も繰り返してしまう「ループ現象」を防ぐための強力な引数です。

| 引数 | 型 | 説明 |
| :--- | :--- | :--- |
| **`no_repeat_ngram_size`** | `int` | 指定された数（N）以上の同じ単語の並び（N-gram）が、文中に二度と出現しないように完全に禁止します。<br>（例: `3` に指定すると、「〜です。〜です。」のような繰り返しが強制的に防がれます） |
| **`repetition_penalty`** | `float` | すでに生成された単語を再度生成する際、その確率にペナルティ（減衰）をかけます。<br>（1.0がペナルティなし。1.2などに設定すると繰り返しが減少します） |

---

## 4. 整形と後処理に関する引数

| 引数 | 型 | デフォルト | 説明 |
| :--- | :--- | :--- | :--- |
| **`clean_up_tokenization_spaces`** | `bool` | `True` | 出力されたテキスト内の不要なスペース（例: `don ' t` $\rightarrow$ `don't`）を自動的に綺麗に整形します。 |
| **`early_stopping`** | `bool` | `False` | ビーム探索時に、すべてのビームが終了トークン（`</s>`など）に達した時点で生成を早期終了し、無駄な計算を省きます。 |

---

## 5. 実用的な指定コード例

以下は、これらの引数を組み合わせて「重複がなく、高品質で引き締まった要約」を出力させるための設定例です。

```python
from transformers import pipeline

summarizer = pipeline("summarization")

text = "（要約したい長いニュースや記事の文章）..."

outputs = summarizer(
    text,
    max_new_tokens=100,              # 最大100トークン生成
    min_new_tokens=30,               # 最低でも30トークン生成
    num_beams=4,                     # ビーム探索を有効化し、探索幅を4にする（高品質化）
    no_repeat_ngram_size=3,          # 3単語以上の同じパターンの繰り返しを完全に禁止
    early_stopping=True,             # 終了条件に達したらすぐに生成を終える
    clean_up_tokenization_spaces=True # 出力スペースの自動整形
)

print(outputs[0]['summary_text'])
```
