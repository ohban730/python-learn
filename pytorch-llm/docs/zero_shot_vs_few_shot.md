# ゼロショット（Zero-shot）の定義とプロンプティング手法

自然言語処理（NLP）やLLM（大規模言語モデル）の分野で頻出する **ゼロショット（Zero-shot）** の意味と、関連する概念（Few-shotなど）について解説します。

---

## 1. ゼロショット（Zero-shot）とは？

ゼロショットとは、モデルに対して **「そのタスクの具体的な回答例（手本）」を一つも与えずに、ぶっつけ本番で指示（タスク）を解かせる手法** のことです。

機械学習において、お手本（訓練データ）を見せることを **「Shot」** と呼ぶため、お見本が「0個」という意味で **Zero-shot** と呼ばれます。

---

## 2. LLMにおけるプロンプトの比較

LLMに入力する「お手本の数」によって、プロンプトの呼び方が変わります。

### ① ゼロショット（Zero-shot）
具体的なお手本を見せず、指示（プロンプト）だけを入力します。

```text
【プロンプト】
入力テキストの感情を「ポジティブ」か「ネガティブ」で分類してください。
テキスト：「この靴は履き心地が最高です！」
分類：
```
* **特徴**: 現代のGPT-4やClaudeなどのLLMは非常に賢いため、このゼロショット形式の指示だけでほとんどのタスクを正しく解くことができます。

### ② ワンショット（One-shot）
お手本（入力と出力のペア）を **1つだけ** 見せてから、本番の問題を入力します。

```text
【プロンプト】
入力テキストの感情を「ポジティブ」か「ネガティブ」で分類してください。

（例）
テキスト：「雨が降って最悪の気分だ。」
分類：ネガティブ

（本番）
テキスト：「この靴は履き心地が最高です！」
分類：
```

### ③ フューショット（Few-shot）
お手本を **いくつか（2〜数個程度）** 見せてから、本番の問題を入力します。

```text
【プロンプト】
入力テキストの感情を「ポジティブ」か「ネガティブ」で分類してください。

（例1）
テキスト：「雨が降って最悪の気分だ。」
分類：ネガティブ

（例2）
テキスト：「映画を見てとても楽しかった！」
分類：ポジティブ

（本番）
テキスト：「この靴は履き心地が最高です！」
分類：
```
* **いつ使うのか？**: 
  * 「JSON形式で出力してほしい」など、出力のフォーマット（形）を厳密に守らせたいとき。
  * 指示（ゼロショット）だけではモデルが意図を理解しにくい、複雑なルールがあるとき。

---

## 3. ゼロショット分類 (Zero-shot Classification) とは？

Hugging Face の `pipeline("zero-shot-classification")` などに代表される技術です。

通常の分類AIは、事前に「スポーツ」「政治」「科学」などのデータで訓練（ファインチューニング）しなければ、そのカテゴリへの分類はできませんでした。

しかし、ゼロショット分類モデル（例：BART-Large-MNLIなど）は、**「事前にそのラベル（カテゴリ）で訓練されていなくても、テキストとラベルの意味の近さを比較することで、後から自由に指定した任意のカテゴリに分類できる」** という画期的な機能を持っています。

```python
from transformers import pipeline

# ゼロショット分類器の作成
classifier = pipeline("zero-shot-classification")

# 分類したいテキスト
sequence_to_classify = "新しいロケットが月に向けて打ち上げられました。"

# 後から自由に決めたカテゴリ（ラベル）
candidate_labels = ["政治", "スポーツ", "宇宙科学", "料理"]

# 実行（モデルは「宇宙科学」のデータで直接訓練されていなくても、意味の近さから正しく分類できます）
result = classifier(sequence_to_classify, candidate_labels)
print(result['labels'][0], result['scores'][0])
# 出力: 宇宙科学 0.98...
```
