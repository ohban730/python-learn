# 複数の事例をまとめてトークン化する（バッチ処理）の仕組みとメリット

書籍の39ページに登場する「複数の事例をトークン化する」という表現について、AIや自然言語処理の文脈における「事例」や「バッチ処理（一括処理）」の意味と、なぜそうするのかを解説します。

---

## 1. 結論：1文ずつではなく「複数の文章をまとめて一括でトークン化する」という意味です

機械学習の分野では、データ1件（1つの文章や1つのツイート）のことを **「事例（Example）」** と呼びます。

したがって、「複数の事例をトークン化する」とは、**「複数の文章（＝事例の集まり、バッチ）をリストの形で受け取り、一挙にまとめてトークン化する」** という処理を指しています。

---

## 2. なぜ「まとめて」処理するのか？（2つの大きなメリット）

1文ずつループで回して処理（`for sentence in corpus:`）するのではなく、まとめて一括処理（バッチ処理）するのには、大きな理由が2つあります。

### メリット①：処理速度が圧倒的に速くなる（並列処理）
Hugging Face の `tokenizers` ライブラリは、裏側が高速なシステム言語である **Rust** で書かれています。
複数の文章（リスト）を一度にトークンナイザーに流し込むと、ライブラリが自動的にPCのCPUをフル活用（マルチスレッド並列処理）して計算を行うため、1万行のデータでも一瞬でトークン化が終わります。

### メリット②：パディング（長さを揃える処理）が正しく機能する
ニューラルネットワークに入力するためには、すべての文章を**「同じ長さ（同じトークン数）」**に揃えた行列（2次元テンソル）にする必要があります。

```text
# 1つのバッチの中に 2つの文章（事例）があるとする

文章1: "I feel great"                ──> トークン数: 3
文章2: "I am feeling very sad today" ──> トークン数: 6

【 まとめて処理（padding=True）すると... 】
「一番長い文章は 6文字だな」と自動判別し、短い文章の後ろを [PAD] で埋めて長さを揃えてくれます。

文章1: [ "I", "feel", "great", "[PAD]", "[PAD]", "[PAD]" ] ──> トークン数: 6
文章2: [ "I", "am", "feeling", "very", "sad", "today" ]    ──> トークン数: 6
```

このように、複数の文章を比較して「その中の最大長」を割り出して一気に埋めるためにも、まとめて処理をする必要があります。

---

## 3. コードの読み解き

書籍のコードを分解して解釈します。

```python
# batch には 複数の事例（データ）が辞書形式で入ってくる
# 例: { "text": ["文章1です", "文章2です", "文章3です"], "label": [0, 1, 0] }
def tokenize(batch):
    # batch["text"] で、テキストのリスト（複数事例）をまとめてトークナイザーに渡している
    return tokenizer(batch["text"], padding=True, truncation=True)
```

---

## 4. `tokenizer` 関数自体は、直接「リスト」を受け取れる？

**結論として、Hugging Faceの `tokenizer` は、最初から「文字列のリスト（複数文）」を直接受け取ることができます。**

### ① 1文だけトークン化する場合 (引数に文字列を渡す)
```python
# 結果は 1次元のリストを含む辞書が返る
result = tokenizer("I feel great")
```

### ② 複数文をまとめてトークン化する場合 (引数にリストを渡す)
```python
# 結果は 2次元のリスト（バッチ）を含む辞書が返る
result = tokenizer(["I feel great", "I am sad"])
```
このように、`tokenizer` 自体に直接リストを渡すだけで自動的に一括処理（バッチ処理）してくれます。

### 🤔 では、なぜわざわざ `tokenize(batch)` という関数を自作したの？

理由は、**データセット（Datasetオブジェクト）の持ち方**にあります。

`emotions` データセットは、ただの「テキストのリスト」ではなく、以下のように `"text"` や `"label"` などの複数の列（カラム）がセットになった **表データ（辞書型）** になっています。

```python
# データセットの構造（イメージ）
{
    "text": ["文章A", "文章B", "文章C"],
    "label": [0, 1, 0]
}
```

このデータセットに対して、`dataset.map()` を使って一括で処理を適用したい場合、データセット全体から**「"text" の列だけを取り出して tokenizer に渡し、得られた結果をデータセットに戻す」**という橋渡し役（ラッパー関数）が必要になります。

そのため、書籍では以下のような `tokenize` 関数を自作して橋渡しをしています。

1. `batch["text"]` でテキストのリストだけを引っこ抜く。
2. それを `tokenizer()` に渡してトークン化する。
3. `dataset.map()` は、この関数の戻り値（`input_ids` など）を自動的にデータセットの新しい列として追加してくれます。
