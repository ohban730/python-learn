# 隠れ状態 (Hidden States) とロジット (Logits) の違い

書籍の44ページ周辺で抽出している **「最後の隠れ状態 (last_hidden_state)」** の正体と、よく混同される **「ロジット (Logits)」** との違いについて解説します。

---

## 1. 結論：最後の隠れ状態は「ロジット」ではありません

これらはモデルが出力する処理フェーズの「異なる段階」のデータを指しています。

* **最後の隠れ状態 (last_hidden_state)**:
  * **正体**: Transformerブロック（エンコーダー）をすべて通過した直後に出力される、**単語や文章の「意味ベクトル」（通常768次元など）**。
  * **意味**: この時点ではまだ「これが何の感情か」という直接的なスコア（分類予測）は含んでおらず、高度な意味情報が凝縮された状態（特徴量）です。
* **ロジット (Logits)**:
  * **正体**: 最後の隠れ状態のベクトルを、さらに「分類ヘッド（Linearレイヤー）」に通した直後の、**「クラスごとの生スコア（例: クラス数である6次元など）」**。
  * **意味**: この数値を Softmax にかけることで、最終的なクラスごとの確率が求まります。

---

## 2. モデル内部の処理フロー図

テキストが入力されてから予測確率が出力されるまでの流れです。

```
 [ 入力テキスト ] ("this is a test")
        │
        ▼ (トークナイザーで数値化)
   [ 単語IDリスト ] (Shape: [1, 6])
        │
        ▼ (Embedding層でベクトル化)
   [ 初期ベクトル ] (Shape: [1, 6, 768])
        │
        ▼ (Transformerエンコーダーを複数層通過)
 🌟[ 最後の隠れ状態 (last_hidden_state) ] (Shape: [1, 6, 768])  <-- ★今回抽出しているのはココ！
        │
        ├───────────────────────────> (外部の別分類器の学習用特徴量として利用)
        ▼ (※ファインチューニング等の場合、ここから分類ヘッドを通す)
   [ 分類ヘッド (Linear層) ]
        │
 🌟[ ロジット (Logits) ] (Shape: [1, 6クラス])                  <-- ★ロジットはココ！
        │
        ▼ (Softmax関数)
   [ クラスごとの予測確率 ] (Shape: [1, 6クラス]、合計が1.0)
```

---

## 3. Shape（テンソル形状）と役割の比較

| 項目 | 最後の隠れ状態 (`last_hidden_state`) | ロジット (`Logits`) |
| :--- | :--- | :--- |
| **形状 (Shape)** | `[batch_size, トークン数, 768]` (例: `[1, 6, 768]`) | `[batch_size, 分類クラス数]` (例: `[1, 6]`) |
| **データの中身** | 768次元の浮動小数点数の並び（意味の空間表現） | 各クラス（悲しい、嬉しい等）の「生の予測得点」 |
| **役割** | テキストの意味を表現する中間層の「特徴量ベクトル」 | 最終的な予測確率を求めるための直前スコア |

---

## 4. なぜ先頭（`[:, 0]`）だけを取り出すのか？

書籍には `outputs.last_hidden_state[:, 0]` というコードが登場します。これは、**「文章全体の意味を代表する1本（768次元）のベクトル」**を取り出すための処理です。

1. `last_hidden_state` の形状は `[1, 6, 768]` であり、6個の単語（トークン）それぞれのベクトルが入っています。
2. BERTやDistilBERTでは、入力テキストの先頭に必ず **`[CLS]`（Classification）** という特殊なトークンが自動で挿入されます。
   * 入力: `"[CLS] this is a test [SEP]"`
3. この `[CLS]` トークンは、Transformerの各層を通過する際、周囲のすべての単語の意味を自己注意（Self-Attention）によって集約するように設計されています。
4. したがって、最後の隠れ状態の**先頭（インデックス 0）である `[:, 0]`** を取り出せば、文章全体を要約した768次元のベクトル（Shape: `[1, 768]`）が得られます。

今回の「特徴抽出アプローチ」では、この取り出した `[CLS]` のベクトル（768次元）を、ロジスティック回帰やランダムフォレストといった外部の分類器に「特徴量」として入力しています。
