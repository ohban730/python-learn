# 質問応答（Question Answering）の2つのアプローチ：抽出型と生成型

Hugging Face の `pipeline("question-answering")` などのタスクにおいて、「最もスコアの高い文中表現を抽出しているだけなのか？」という疑問について解説します。

---

## 1. 結論：モデルの種類によって異なります

質問応答（QA）タスクには、大きく分けて **「抽出型」** と **「生成型」** の2種類のアプローチがあります。

* **BERTなどのモデル（抽出型）の場合**
  * **はい、その通りです。** 本文（コンテキスト）の中から、最も答えとして確率（スコア）が高い「開始位置」と「終了位置」を見つけ、**その範囲の文字列をそのまま抜き出しているだけ**です。
* **GPTやT5などのモデル（生成型）の場合**
  * いいえ、単なる抽出ではありません。本文の内容を読み込んだ上で、**自分の言葉で新しい文章を「生成」**して回答します。

---

## 2. 抽出型質問応答 (Extractive QA) の仕組み

BERTやRoBERTaなど、主に**エンコーダー型**のモデルが使われます（Hugging Face のデフォルトの `pipeline("question-answering")` もこれに該当します）。

```
【入力】
  質問： 「アインシュタインの生まれた国は？」
  本文： 「アルベルト・アインシュタインは1879年にドイツ帝国で生まれた。」
  
【モデルの処理】
  本文の各単語（トークン）に対して、モデルが以下の2つのスコアを計算します。
    1. ここから答えが始まる確率（Start Score）
    2. ここで答えが終わる確率（End Score）
  
  [アルベルト] [・] [アインシュタイン] ... [ドイツ帝国] [で] [生まれた]
  Start: 0.01   0.0   0.0          ...  0.98        0.0  0.0
  End:   0.0    0.0   0.0          ...  0.97        0.0  0.0

【出力】
  Start確率 × End確率 が最も高くなる範囲「ドイツ帝国」を切り出して出力する。
```

### 特徴と限界
* **本文に書かれている言葉しか出力できません。**
* たとえ「ドイツ帝国」が正解だとしても、本文に「ドイツ」としか書かれていなければ、出力は「ドイツ」になります。
* 言い換えや、要約、気の利いた会話風の返答（「答えはドイツです」など）は一切できません。

---

## 3. 生成型質問応答 (Generative / Abstractive QA) の仕組み

GPTシリーズ（デコーダー型）やT5（エンコーダー・デコーダー型）など、主に**テキスト生成能力を持つモデル**が使われます。

```
【入力】
  ユーザー： 「以下の本文を元に、アインシュタインの生まれた国を教えてください。
             本文：アルベルト・アインシュタインは1879年にドイツ帝国で生まれた。」

【モデルの処理】
  本文と質問を丸ごと読み込み、次の単語予測を繰り返して文章を生成します。

【出力】
  「アインシュタインが生まれた国はドイツ帝国です。」
```

### 特徴とメリット
* 本文にない表現を使って、分かりやすく言い換えたり、要約したりできます。
* **ハルシネーション（事実に基づかない嘘）**を出力してしまうリスクがあります（抽出型は本文から抜き出すだけなので、モデルが嘘の単語を捏造する心配はありません）。

---

## 4. 抽出型と生成型の比較

| 項目 | 抽出型 QA (Extractive) | 生成型 QA (Generative) |
| :--- | :--- | :--- |
| **代表的なモデル** | BERT, RoBERTa, LUKE | GPT-4, Llama, T5 |
| **処理内容** | 本文のスライス（切り出し） | テキスト生成（作文） |
| **答えの柔軟性** | ❌ 本文の文字列そのまま（一字一句同じ） | ⭕ 自由な文章、要約、会話風の回答 |
| **信頼性（嘘のなさ）** | ⭕ 極めて高い（本文にない単語は出せない） | ❌ 嘘の事実（ハルシネーション）を語るリスクあり |
| **計算コスト** | ⭕ 非常に軽量で高速（数ミリ秒〜数十ミリ秒） | ❌ 重い（文章生成のループが回るため時間がかかる） |
