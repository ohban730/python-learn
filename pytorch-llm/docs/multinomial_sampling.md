# torch.multinomial と確率比例サンプリングの数理

書籍161ページにおいて、テキスト生成の「決定論的（常に一番高い確率の単語を選ぶ）」な戦略から、**「確率的（確率スコアに応じてランダムに選ぶ）」** な戦略へ切り替えるために、**`torch.multinomial`** 関数が登場します。

「確率スコアに比例する」とはどういうことなのか、そしてなぜLLMの文章生成（ChatGPTなどの応答の揺らぎ）においてこの関数が必要不可欠なのかを詳しく解説します。

---

## 1. `torch.multinomial` とは何か？

`torch.multinomial` は、**多項分布（Multinomial Distribution）** に基づいて、与えられたテンソル（確率配列）の数値の大きさに比例する確率で、インデックスをランダムにサンプリング（抽選）する関数です。

イメージとしては、**「各単語の確率に応じた面積を持つルーレット盤を作って、それを回して矢が刺さった単語を選ぶ」** という抽選機です。

---

## 2. argmax（決定論）と multinomial（確率比例）の対比

次の単語の予測候補として、Softmaxから以下の確率が出力されたとします。

*   `"forward"`: **70%** (0.7)
*   `"toward"` : **20%** (0.2)
*   `"closer"` : **10%** (0.1)
*   `"pizza"`  : **0%**  (0.0)

それぞれの関数でサンプリングを行った場合の挙動の違いです。

### 🎯 torch.argmax の場合
最も確率が高い `"forward"`（インデックス値）を**常に100%選択**します。
何万回実行しても、出力結果は常に `"forward"` であり、一切変化しません。

### 🎡 torch.multinomial の場合
確率の比率（70% : 20% : 10%）に従って、以下の面積を持つルーレット盤を回します。

```text
  【確率比例ルーレット盤】
  ┌─────────────────────────────────────────┐
  │                 forward                 │  <── 70%の広さ
  │                 (70%)                   │
  ├──────────────┬──────────────────────────┤
  │    toward    │          closer          │  <── 20% と 10% の広さ
  │    (20%)     │          (10%)           │
  └──────────────┴──────────────────────────┘
  ※ "pizza" (0%) は的（エリア）が存在しません。
```

このルーレットを回して抽選するため：
*   **約 70%** の頻度で最も確率が高い `"forward"` が選ばれます。
*   しかし、**約 20%** の頻度で `"toward"` が選ばれることもあります。
*   さらに、**約 10%** の頻度で意外な単語である `"closer"` が選ばれる可能性もあります。
*   確率が 0% の `"pizza"` は、的が存在しないため、**何万回回しても絶対に選ばれません**。

---

## 3. なぜLLMの文章生成にこれが必須なのか？

### ❌ 常に最も確率が高い単語を選ぶ (argmax) デメリット
`argmax` のみでテキストを生成する戦略を「貪欲法 (Greedy Decoding)」と呼びます。
Greedy Decoding を行うと、**「同じ質問（プロンプト）を入力したら、AIは毎回1文字も違わずに全く同じ回答を出力する」** ことになります。

これでは人間味（会話の揺らぎや多様性）がなく、何回質問しても新しいアイデアや回答のバリエーションが得られない、機械的なシステムになってしまいます。

### 🌟 確率比例サンプリング (multinomial) のメリット
`multinomial` を使うことで、モデル自身が算出した確率のバランスを保ったまま、適度な「揺らぎ」を文章生成に与えることができます。
これにより、同じプロンプトを入力しても、**実行するたびに少しずつ異なる、自然で人間らしい創造的なテキスト**が生成されるようになります。

> [!NOTE]
> 現代のLLM（ChatGPTやClaudeなど）のAPIに存在する **`Temperature（温度）`** や **`Top-k`**, **`Top-p（Nucleus）`** といったパラメータは、この `torch.multinomial` に渡す前段階の確率分布（ルーレットの面積の比率）を調整・変形するための技術です。最終的な「単語の決定」は、調整後の確率をこの `torch.multinomial` に通してサンプリングしています。

---

## 4. PyTorchコード上の使い方と主要引数

```python
next_token_id = torch.multinomial(probas, num_samples=1)
```

*   **`input` (第1引数 / `probas`)**:
    サンプリング元の確率が入ったテンソル。合計値が `1.0` (100%) になっている必要があります（通常は `softmax` を通したデータ）。
*   **`num_samples`**:
    何回抽選して値を取り出すかの回数。LLMで「次の単語を1つだけ選ぶ」時は `1` を指定します。
*   **`replacement` (デフォルトは `False`)**:
    「重複して引くことを許すか（復元抽出か）」の設定。
    *   `False` (非復元抽出): 一度引いたインデックスは除外されます。
    *   `True` (復元抽出): 引いたインデックスを元に戻すため、同じものが重複して選ばれる可能性があります。
    *   ※ `num_samples=1` の場合はどちらでも挙動は同じですが、複数トークンを同時サンプリングする際は `True` に設定する必要があります。
