# エンコーダー（BERT）とデコーダー（GPT）の違いと事前学習タスクの関係

「BERT（エンコーダー）＝ 穴埋めタスク」と「GPT（デコーダー）＝ 次単語予測タスク」がどう関係しているのかは、**「未来の単語（右側の情報）を見てよいかどうか」というルール（Attention マスク）**を理解すると、すっきりと腹に落ちます。

---

## 1. 決定的な違いは「Attention の見え方（マスク）」

Transformerの心臓部である **Self-Attention（自己注意機構）** において、情報の見え方に制限をかけるかどうかが、エンコーダーとデコーダーの最大の違いです。

```
【エンコーダー（双方向型 / BERT）】
全方向を見渡せる（カンニングOK）

   [私]  <─>  [は]  <─>  [?]  <─>  [を]  <─>  [食べた]
   （すべての単語が、他のすべての単語の情報を参照できる）

──────────────────────────────────────────────

【デコーダー（因果的・単方向型 / GPT）】
自分より左側（過去）しか見えない（未来のカンニング禁止）

   [私] ──> [は] ──> [リンゴ] ──> [を] ──> [?] (未来は見えない)
   （右側にある「未来の単語」へのリンクはすべてマスクされて遮断される）
```

---

## 2. なぜそのタスクに向いているのか？

### ① BERT（エンコーダー）が「穴埋めタスク」に向いている理由
BERTの事前学習タスクは **Masked Language Modeling（マスク化言語モデル）** と呼ばれる「穴埋め問題」です。

* **タスク**: `私は [MASK] を食べた。` の `[MASK]` に入る単語を予測する。
* **なぜエンコーダーなのか**:
  穴埋めを解くには、`[MASK]` の**左側（私は）**と**右側（を食べた。）**の両方のヒントを合体させて考える必要があります。
  左右すべての文脈を同時に見渡せる「双方向（エンコーダー）」の仕組みだからこそ、この穴埋めタスクを解くことができ、結果として文章全体の意味を深く理解する（エンコードする）能力が身につきます。
* **弱点**:
  未来の単語（右側）が見えてしまっているため、文章を「左から右へ1語ずつ生成していく」というタスクには使えません（未来をカンニングできてしまうため、学習が破綻します）。

### ② GPT（デコーダー）が「次単語予測タスク」に向いている理由
GPTの事前学習タスクは **Causal Language Modeling（因果的言語モデル）** と呼ばれる「次単語予測」です。

* **タスク**: `私は` -> `リンゴを` -> `[?]` と順に入力されたとき、次に続く単語（`食べた`）を予測する。
* **なぜデコーダーなのか**:
  文章を生成するためには、「自分の左側にある単語だけを見て、右側に新しい単語を1つずつ生み出す」必要があります。
  そのため、Attentionの計算において**「未来の単語（右側）を見えなくするマスク（因果マスク / Causal Mask）」**を強制します。
  この「カンニング禁止」のルールがあるからこそ、モデルは「出力した単語を次の入力にして、さらにその次を予測する」というループを回すことができ、自由なテキスト生成（デコード）が可能になります。

---

## 3. なぜ「エンコーダー」「デコーダー」と呼ばれるのか？（語源）

元々のTransformerは、機械翻訳（Seq2Seq）のために開発されました。そのときの役割分担が名前の由来です。

```
【翻訳のプロセス (例: 英語から日本語へ)】

「I love cats.」を入力
      │
      ▼ (理解する)
【エンコーダー (BERTの先祖)】
英文全体（左右両方向）を読み込み、意味の「数値ベクトル」に変換（符号化 / エンコード）する。
      │
      ▼ (生成する)
【デコーダー (GPTの先祖)】
その数値ベクトルを受け取り、過去に出力した日本語だけを見ながら、「私は」->「猫が」->「大好きです」と、左から右へ順番に出力（復号化 / デコード）する。
```

* **エンコーダー（BERT）**は、入力された文章を「理解し、数値ベクトルに落とし込む（エンコードする）」のが得意なため、文章分類や感情分析、検索などの**「文章理解タスク」**に向いています。
* **デコーダー（GPT）**は、数値から文章を「新しく紡ぎ出す（デコードする）」のが得意なため、チャットボットや対話、文章執筆などの**「文章生成タスク」**に向いています。
