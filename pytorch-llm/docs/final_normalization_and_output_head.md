# 最後の層正規化 (Final LayerNorm) と線形出力層 (LM Head) の役割

書籍123ページの「図4-15：GPTモデルのアーキテクチャの概要図」の最上部には、トランスフォーマーブロックを通過した後に以下の2つのコンポーネントが配置されています。

1.  **最後のLayerNorm (Final LayerNorm)**
2.  **線形出力層 (Linear Output Head / LM Head)**

モデルの計算の最終段階（出口）に位置するこれら2つのパーツが、どのような仕組みで動作し、いかにして「内部ベクトル」から「次の単語の予測確率」へとデータを変換しているのかについて詳しく解説します。

---

## 1. 最後の層正規化 (Final LayerNorm / `self.final_norm`)

### 🛠️ 何をしているのか？
すべてのトランスフォーマーブロック（GPT-2 124Mモデルなら12層）の計算を完全に終えた直後のテンソル（Shape: `[Batch, Seq_Len, 768]`）に対して、最終的な仕上げとして適用する LayerNorm（層正規化）です。

```python
# DummyGPTModel の実装より (C:\Users\owner\Documents\lab\llm-sandbox\chap04\main.py)
self.final_norm = LayerNorm(cfg["emb_dim"])
```

### ❓ なぜ必要なのか？（数理的役割）
トランスフォーマーブロックを巡る旅の中で、データはアテンションやFFNの「ショートカット接続（残差加算）」を何度も通過します（12層モデルなら、加算が24回繰り返されます）。

足し算が何度も繰り返されることで、ベクトルの数値（スケール）は徐々に大きくなり、値のばらつき（分散）が偏って歪んでしまいます。
この不安定な数値スケールのまま次の全結合層（Linear）に入力してしまうと、**特定の要素だけが極端に巨大な値になってしまい、予測の精度が低下したり、数値計算上のエラーが発生したりします。**

そのため、最後の全結合層の直前で、**「もう一度ベクトルの平均を0、分散を1に綺麗に標準化（整頓）し、最後の予測器が最も安定して動作できるようにする」**のが、この最後のLayerNormの役割です。

---

## 2. 線形出力層 (Linear Output Head / LM Head / `self.out_head`)

### 🛠️ 何をしているのか？
最後のLayerNormで綺麗に標準化された埋め込み次元（768次元）のベクトルを、**「モデルが知っているすべての単語（語彙数: 50,257個）」の次元へと一気に引き伸ばす（射影する）全結合層（Linear層）**です。

```python
# DummyGPTModel の実装より
self.out_head = nn.Linear(cfg["emb_dim"], cfg["vocab_size"], bias=False)
```

### ❓ なぜ必要なのか？（予測の数理）
モデルが最終的にやりたいタスクは、**「50,257個の英単語の中から、次にどの単語が来るかを当てる『50,257クラスの分類問題』」**です。

しかし、モデルの内部を流れるデータの次元数は、アテンションやFFNで効率よく計算するために `768次元` に抑えられています。768個の数字のままでは、50,257語のどれに対応するのか判定することができません。

そこで、この `nn.Linear`（重み行列のサイズ: `[50257, 768]`）を使用し、行列の掛け算（行列積）を行って次元を変換します。

$$\text{出力ロジット} = X_{\text{final\_norm}} \cdot W_{\text{out\_head}}^T$$

*   **入力の形状**: `[Batch, Seq_Len, 768]`
*   **重みの形状**: `[50257, 768]` (転置して `[768, 50257]` として掛け算)
*   **出力の形状**: `[Batch, Seq_Len, 50257]` (4×50,257次元のテンソルへ引き伸ばされる！)

これによって、入力された各トークン（位置）に対して、**「50,257個の英単語それぞれに対する、次に来るふさわしさスコア（ロジット）」** が算出されます。

---

## 3. 最終出力：ロジットから次の単語の予測確率へ

線形出力層から出力された 50,257個の数値（**ロジット**）は、まだ単なる強さのスコア（プラスやマイナスの実数）です。

これを、人が見て理解できる、かつプログラムで扱いやすい「確率」へと変換するのが最後のステップです。

```text
    線形出力層 (Linear)  ──>  50,257次元のスコア（ロジット）
                                    │
                                    ▼
     【 Softmax 関数 】  ──>  0.0 〜 1.0 (合計が 100%) の「確率」へ変換
                                    │
                                    ▼
       【 サンプリング 】  ──>  最も確率の高い単語（または確率に基づいたランダム）
                                を選んで、次の単語（例: "forward"）を出力！
```

### 💎 まとめ
*   **最後のLayerNorm**は、加算で歪んだデータを最終予測の前にきれいに「おめかし（標準化）」する。
*   **線形出力層（LM Head）**は、モデルの内部的な意味表現（768次元）を、実際の単語予測（50,257語のどれか）へと「翻訳（拡大投影）」する。

この2つがコンビで動くことで、Transformerブロックの抽象的な計算が、具体的な「テキスト生成」という最終結果へと結びついているのです。
