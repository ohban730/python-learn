# ロジスティック回帰における「多クラス分類」と「シグモイド関数」の仕組み

書籍の47〜48ページに登場する「ロジスティック回帰モデル」について、その心臓部である **シグモイド関数との関係性**、および3つ以上の選択肢（多クラス）を分類する裏側の仕組みを解説します。

---

## 1. シグモイド関数とロジスティック回帰の関係

結論から言うと、**シグモイド関数は「ロジスティック回帰モデルの心臓部（コアエンジン）」** です。

ロジスティック回帰の計算は、内部で以下の **2ステップ** で行われています。

```
 [ 入力データ (x) ]
        │
        ▼ 【ステップ1】重み(w)を掛けて足し算する (普通の線形計算)
   生スコア (z) = w1*x1 + w2*x2 + ... + b   (範囲: -∞ 〜 +∞)
        │
        ▼ 【ステップ2】★ シグモイド関数に通す ★
   確率 (p) = σ(z) = 1 / (1 + e^-z)        (範囲: 0.0 〜 1.0)
```

### ① なぜ「回帰」という名前なのに分類なのか？
* **線形回帰 (Linear Regression)**:
  ステップ1の生スコア $z$（例: 売上予測などの数値）をそのまま出力するモデルです。
* **ロジスティック回帰 (Logistic Regression)**:
  ステップ1の生スコア $z$ を、**シグモイド関数（別名: ロジスティック関数）** に通すことで強制的に `0.0` 〜 `1.0`（0%〜100%）の「確率」に変換し、分類を行うモデルです。

つまり、**「線形回帰 ＋ シグモイド関数 ＝ ロジスティック回帰」** という関係になっています。

### ② ニューラルネットワークとの関係
ロジスティック回帰は、ディープラーニングにおける **「1層の全結合層（`nn.Linear`）＋ シグモイド関数（`F.sigmoid`）」** と数学的に完全一致します。

---

## 2. 疑問の出発点：ロジスティック回帰は本来「2択専用」

ロジスティック回帰（Logistic Regression）は、上記の通りシグモイド関数を使うため、本来 **「二値分類（2択）」** のためのアルゴリズムです。

* **2択の例**: 「迷惑メールか / 正常メールか」「病気があるか / ないか」
* **判定方法**: 出力が `0.5`（50%）以上なら「陽性(1)」、未満なら「陰性(0)」と判定します。

しかし、今回の感情分類データセット（`emotions`）は、**「悲しみ, 喜び, 愛, 怒り, 恐れ, 驚き」の 6つのクラス（6択）** が存在します。このように分類先が3つ以上あるタスクを **「多クラス分類 (Multi-class Classification)」** と呼びます。

---

## 3. 2択モデルで「多クラス（6択）」を分類する 2つの仕組み

本来2択しかできないロジスティック回帰を多クラス対応させるため、裏側では主に以下の2つの拡張手法が使われています。

### アプローチ①：ソフトマックス回帰（Multinomial / Softmax Regression）★現代の標準
シグモイド関数の代わりに、**ソフトマックス関数（Softmax）** を使用します。

```text
 【 入力ベクトル (768次元) 】
             │
             ▼ (6クラス分それぞれのスコアを計算)
   [ 悲しみ: 3.2,  喜び: -1.0,  愛: 0.5,  怒り: 1.1,  恐れ: 0.2,  驚き: -2.0 ]
             │
             ▼ (Softmax 関数を通す)
   [ 悲しみ: 80%,  喜び: 1%,    愛: 5%,   怒り: 10%,  恐れ: 3%,   驚き: 1% ]
                                                             (合計 100%)
```

* クラスの数（今回なら6個）だけ同時にスコアを計算し、Softmax を使って **「すべてのクラスの確率の合計が 1.0 (100%)」になる確率分布** を一括で算出します。
* ※これは、PyTorch や Transformer などのディープラーニングにおける出力層（`Linear` ＋ `Softmax`）と完全に同じ数学的構造です。

---

### アプローチ②：One-vs-Rest (OvR / 一対他アプローチ)
2択のロジスティック回帰モデルを、**クラスの数だけ（今回なら6個）並列して作る**手法です。

1. **モデル1**: 「『悲しみ』か？ / 『それ以外』か？」を判定
2. **モデル2**: 「『喜び』か？ / 『それ以外』か？」を判定
... （以下6個分）

新しい文章を入力したとき、6個のモデルすべてに判定させ、**「最も高い確率（確信度）を出したモデルのクラス」**を最終結果として採用します。

---

## 4. scikit-learn での自動制御

Python の `scikit-learn` ライブラリの `LogisticRegression` は、非常に賢く作られています。

```python
from sklearn.linear_model import LogisticRegression

lr_clf = LogisticRegression(max_iter=3000)
lr_clf.fit(X_train, y_train) # y_train に 0~5 の数値が入っている
```

私たちが特別なコードを書かなくても、`y_train` の中に `0, 1, 2, 3, 4, 5` といった3種類以上のラベルが入っていることを自動的に検出し、**裏側で自動的に多クラス分類モード（デフォルトはソフトマックス回帰）へ切り替えて学習**を行ってくれます。
