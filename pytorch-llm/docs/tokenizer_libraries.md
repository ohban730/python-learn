# トークナイザー（Tokenizer）ライブラリの種類と違い

自然言語処理（NLP）における「トークナイザー（テキストを単語や数値トークンに分割するプログラム）」のライブラリについて、Hugging Face以外の主要なライブラリや、`tokenizer` という別プロジェクトの有無を解説します。

---

## 1. Hugging Face 以外にトークナイザーはある？

**はい、非常に多くの主要なトークナイザーライブラリが存在します。**

Hugging Face の `tokenizers`（Rustで書かれた超高速なライブラリ）は現在のディープラーニング界隈のデファクトスタンダード（標準）となっていますが、それ以外にも、業界を支える巨頭ライブラリがいくつかあります。

### ① SentencePiece (by Google)
* **概要**: Googleが開発した、C++実装（Pythonからも利用可能）のオープンソース・トークナイザーです。
* **特徴**:
  * Hugging Face 登場以前から存在する、現代のサブワード・トークナイズの基礎を築いたライブラリです。
  * **事前学習用のテキストをそのまま渡すだけで、単語の区切り（スペース）がない日本語や中国語でも、自動で最適なサブワードに分割する辞書を作成**してくれます。
  * **主な使用例**: T5、Llama（Meta）、Alpaca などの超有名モデルの裏側で、Hugging Faceを通さずに SentencePiece が直接トークン分割を行っています。

### ② Tiktoken (by OpenAI)
* **概要**: OpenAIが自社のGPT-3.5やGPT-4、ChatGPTなどのために独自開発した、超高速なBPE（Byte Pair Encoding）トークナイザーライブラリです。
* **特徴**:
  * `pip install tiktoken` で簡単に使えます。
  * Rustで実装されており、Hugging Faceのトークナイザーよりもさらに高速に動作します。
  * OpenAIのAPIに入力するトークン数を正確にカウントしたり、テキストをエンコード・デコードするのに必須のライブラリです。

### ③ 特定言語専用の形態素解析器（日本語など）
日本語などの言語では、サブワード（単語の一部）に分割する前に、文法的に「意味のある単語（形態素）」に分ける必要があります。そのため、以下の解析器が「トークナイザー」として使用されます。
* **MeCab (みかぶ)**: 日本語NLPの超定番。Hugging Faceの日本語モデルの多くも、内部でMeCab（Python用の `fugashi` を経由）を使ってトークナイズしています。
* **Sudachi (すだち)**: 現代的な商業利用に強い日本語解析器。
* **Spacy / NLTK**: 英語をはじめとする多言語の伝統的なNLPで使われる、ルールベースの高速トークナイザーを内包した統合ライブラリ。

---

## 2. `tokenizer` という名前の別ライブラリはある？

Pythonのパッケージ管理システム (PyPI) には、以下のような名前のパッケージが登録されています。

* **`tokenizers` (Hugging Face製)**
  * これが最も有名で、通常開発者が「tokenizersライブラリ」と呼ぶものです。
* **`tokenizer` (別プロジェクト)**
  * PyPIに `tokenizer` という単数形の名前のライブラリもいくつか存在します（例: アイスランド語やラテン語などの特定言語用の簡易トークン分割ツールや、古いルールベースの個人用スクリプト）。
  * しかし、これらは汎用的なLLMや深層学習で使われるものではありません。

### 混乱しやすいポイント
Hugging Face のエコシステムの中だけでも、似た名前が2つあり、それぞれ役割が異なります。

```
【 似ているが別物の Hugging Face ライブラリ 】

  1. tokenizers (Pythonライブラリ)
     * Rustで実装された、テキストの分割と数値化を行う「エンジン単体」の低レイヤーライブラリ。
     * pip install tokenizers

  2. transformers.AutoTokenizer (transformersライブラリ内のモジュール)
     * Hugging Face Hubから任意のモデル用トークナイザーをダウンロードして使うための高レイヤーAPI。
     * 内部で上記の「tokenizers」エンジンを呼び出しています。
```

---

## 3. まとめ

* Hugging Face 以外でLLMによく使われるトークナイザーは、**Googleの `SentencePiece`** と **OpenAIの `tiktoken`** です。
* 単数形の `tokenizer` というライブラリもありますが、実務や深層学習で使われることはほぼありません。
