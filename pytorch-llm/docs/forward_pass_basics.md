# フォワードパス (Forward Pass / 順伝播) の仕組み

書籍の55ページに登場する「フォワードパスでラベルを渡すと…」という表現について、**フォワードパスの正確な意味**と**バックワードパスとの違い**を解説します。

---

## 1. 結論：フォワードパス（順伝播）とは？

フォワードパス（Forward Pass / 順伝播）とは、**入力データをモデルの先頭から通過させ、最後の出力（予測結果や損失）まで「順番に順方向に計算を進める処理」** のことです。

PyTorch や Transformers において、`output = model(**inputs)` のようにモデルにデータを入力して結果を受け取るコードを実行する動作そのものを指します。

```text
 【 フォワードパス (Forward Pass) の流れる向き 】

  [入力データ] ──> [埋め込み層] ──> [Attention層] ──> [全結合層] ──> [予測結果 (Logits)] ──> [損失 (Loss)]
  (input_ids)                                                                            (誤差)
  ─────────────────────────────────────────────────────────────────────────────────────────────>
                                      順方向 (左から右へ計算)
```

---

## 2. フォワードパス vs バックワードパスの比較

機械学習（ディープラーニング）の学習プロセスは、**「フォワードパス」** と **「バックワードパス」** の対セットで動いています。

| 項目 | フォワードパス (Forward Pass / 順伝播) | バックワードパス (Backward Pass / 逆伝播) |
| :--- | :--- | :--- |
| **計算の向き** | **入力 ──> 出力 (順方向)** | **損失 ──> 入力 (逆方向)** |
| **主な目的** | 予測値（Logits）や損失（Loss）の計算 | 誤差の傾き（勾配）を計算し、モデルの重み $W$ を修正 |
| **実行タイミング**| 推論（予測）の時と、学習の最初 | 学習の時のみ（`loss.backward()` の実行時） |
| **PyTorchでのコード** | `output = model(**inputs)` | `loss.backward()` |

---

## 3. 書籍の記述「フォワードパスでラベルを渡すと…」の意味

書籍55ページの該当コード：

```python
output = model(**inputs)
```

通常、Hugging Face のモデルに `input_ids` や `attention_mask` だけを渡してフォワードパスを実行すると、モデルは予測確率の元となる **`output.logits`**（予測結果）だけを出力します。

しかし、入力の中に **`labels`（正解ラベル）も含めて** フォワードパスを実行すると、モデル内部の `forward()` メソッドが「予測結果の計算」に続けて「正解との誤差（損失 CrossEntropyLoss）の計算」まで全自動で行い、**`output.loss`** として一緒に返してくれるようになります。

```text
 ■ labels なしのフォワードパス:
   inputs ──> [ Model ] ──> output.logits (予測結果のみ)

 ■ labels ありのフォワードパス (書籍の記述):
   inputs + labels ──> [ Model ] ──> output.logits (予測結果) + output.loss (自動計算された損失)
```

---

## 4. まとめ

* **フォワードパス**: データをモデルに読み込ませて、予測結果や損失を計算する「順方向の実行」のこと。
* **PyTorchでの実体**: `model(inputs)` や `model.forward(inputs)` を実行すること。
* 推論（テスト）の時は、重みを更新する必要がないため、**フォワードパスだけを実行**します。
