# Google翻訳の歴史的変遷とGoogleの重要モデル「T5」

「Google翻訳の裏側の進化」と、Googleが開発した自然言語処理の歴史的モデル「**T5**」について解説します。

---

## 1. Google 翻訳の裏側はどのように変わってきたか？

Google翻訳のエンジンは、時代とともに劇的な進化を遂げています。

```
【 統計的機械翻訳 (SMT) 】（〜2016年以前）
  * 単語やフレーズをパズルのように機械的に置き換える。
  * 「直訳」感が強く、文法がめちゃくちゃになりがちだった。
        │
        ▼
【 ニューラル機械翻訳 (GNMT) 】（2016年〜）
  * Googleが「GNMT」を発表。**Seq2Seq + LSTM（RNNの一種）**を採用。
  * 文全体の流れを捉えた翻訳が可能になり、翻訳精度が劇的に向上し世界中に衝撃を与えた。
        │
        ▼
【 Transformerベースモデル 】（2020年頃〜）
  * LSTM（時間がかかる）から、並列処理が得意な **Transformer** をベースにした多言語翻訳モデルへと移行。
        │
        ▼
【 LLM（Geminiなど）の技術統合 】（現在）
  * 単なる直訳を超えて、前後の長い文脈やスラング、専門知識を踏まえた超高度な翻訳に進化。
```

---

## 2. Google の重要モデル「T5」とは？

**T5 (Text-to-Text Transfer Transformer)** は、2019年末にGoogleが発表した、現代のLLM（GPTなど）の直接的なアイデアの基礎となった極めて重要なモデルです。

### 最大の特徴：「すべてのタスクをテキスト変換にする」
T5が登場する前、AIモデルはタスクごとに「出力の形」が全く異なっていました。
* 分類タスク ──> 出力は「0 か 1」（数値）
* 翻訳タスク ──> 出力は「テキスト」

Googleはこれに対し、**「すべての自然言語処理タスクは、テキストを入力して、テキストを出力する（Text-to-Text）という同じ形で処理できるはずだ」**と考えました。

```
【T5 のすべての処理イメージ】

   [入力テキスト（タスク指示付き）]                 [出力テキスト]
   
   "translate English to German: That is good."  ──>  "Das ist gut."
   
   "sst2 sentence: I love coding."               ──>  "positive"
   
   "summarize: (長いニュース記事)..."            ──>  "(要約文)"
```

### なぜ T5 がそこまで重要なのか？
1. **プロンプトエンジニアリングの元祖**
   * 「`translate English to German:`」や「`summarize:`」のような**タスク指示（プレフィックス）**を入力に含めてモデルの挙動をコントロールするやり方は、現在のChatGPTなどで行う「プロンプトによる指示」の直接的な原型です。
2. **モデル構造**
   * BERT（エンコーダーのみ）やGPT（デコーダーのみ）と異なり、T5はオリジナルのTransformerと同じ **「Encoder-Decoder」両方を持つ構造** をしています。
   * 入力をエンコーダーで完全に理解し、デコーダーで回答のテキストを生成するため、翻訳や要約のようなタスクで非常に高いパフォーマンスを発揮します。
3. **今でも大活躍している**
   * T5の日本語特化モデル（`megagonlabs/t5-base-japanese-web` など）は、現在でも「高精度かつ軽量な翻訳・要約・文書生成エンジン」として、多くの日本国内の開発現場で無料で活用されています。
