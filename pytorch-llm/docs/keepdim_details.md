# keepdim=True/False のテンソル形状とブロードキャストの視覚的図解

平均（`mean`）や合計（`sum`）を求める際、次元を消滅させる `keepdim=False` と、次元の構造を維持する `keepdim=True` の違いについて、グラフィカルなテキストアートで視覚的に解説します。

---

## 1. 2次元行列の定義 (Shape: [2, 3])

例として、以下の `2行 3列` のテンソル `matrix` を考えます。

```text
matrix (2次元行列)
Shape: [2, 3]
┌───────────────┐
│ 1.0  2.0  3.0 │  <-- 0行目
├───────────────┤
│ 4.0  5.0  6.0 │  <-- 1行目
└───────────────┘
```

このテンソルの横方向（`dim=-1`）の平均を計算します（0行目の平均は `2.0`、1行目の平均は `5.0` です）。

---

## 2. keepdimの有無による「形状（Shape）」の違い

### ❌ A. keepdim=False (デフォルト) の場合
要素数は `2` に縮小され、**次元（縦・横の軸の概念）そのものが消滅**します。形状は `[2]` という「1次元のフラットなリスト」になります。

```text
Shape: [2]
┌───────────┐
│ 2.0 │ 5.0 │
└───────────┘
※ 縦や横という概念を失い、単に値が2つ並んでいるだけの「1次元の棒」になります。
```

### ⭕ B. keepdim=True の場合
要素数は `1` に縮小されますが、**元の「2行」という縦構造（次元）をキープ**します。形状は `[2, 1]` という「2行 1列のビル型（2次元）」になります。

```text
Shape: [2, 1]
┌───────┐
│  2.0  │  <-- 0行目の平均
├───────┤
│  5.0  │  <-- 1行目の平均
└───────┘
※ 要素の部屋は1列分しかありませんが、元の2次元の「縦の並び構造」が維持されています。
```

---

## 3. なぜ keepdim=True が必要なのか？ (ブロードキャストの数理)

平均を引いてデータを標準化する処理（`matrix - mean`）をするときに、この形状の違いが結果を左右します。

### ⭕ A. keepdim=True の場合
PyTorchの**ブロードキャスト（自動拡張）機能**が完璧に機能します。
`[2, 3]` から `[2, 1]` を引く際、PyTorchは「足りない列数(3)の分だけ、平均値(2.0と5.0)を横にコピペ拡張して引き算すればよい」と一意に解釈できます。

```text
  [元の行列: 2x3]           [平均の自動拡張: 2x3]            [計算結果: 2x3]
  ┌───────────────┐         ┌───────────────┐         ┌───────────────┐
  │ 1.0  2.0  3.0 │         │ 2.0  2.0  2.0 │         │ -1.0  0.0  1.0 │
  ├───────────────┤    -    ├───────────────┤    =    ├───────────────┤
  │ 4.0  5.0  6.0 │         │ 5.0  5.0  5.0 │         │ -1.0  0.0  1.0 │
  └───────────────┘         └───────────────┘         └───────────────┘
                                                       --> 【成功】
```

### ❌ B. keepdim=False の場合
`[2, 3]` から `[2]` を引こうとします。
引く側の `[2]` はただのフラットなリスト（縦横がない）です。PyTorchは、この `2` 個の数値を「縦方向に並べるべきか」「横方向に並べるべきか」のルールを決定できません。

```text
  [元の行列: 2x3]           [平均: 2]
  ┌───────────────┐         ┌───────────┐
  │ 1.0  2.0  3.0 │         │ 2.0 │ 5.0 │
  ├───────────────┤    -    └───────────┘
  │ 4.0  5.0  6.0 │         (縦横の対応がつかず、自動拡張できない！)
  └───────────────┘
                             --> 【RuntimeErrorでクラッシュ！】
```

このため、PyTorchはエラーを出力して計算を拒否します。
