# ステミング（Stemming）と見出し語化（Lemmatization）の違い

書籍の35ページに登場する、単語を基本形に統一するテキスト前処理技術である **ステミング** と **見出し語化** について、その概要、仕組みの違い、および現代のLLMにおける扱いを解説します。

---

## 1. 共通の目的：単語の「正規化（Normalization）」

英語などの言語では、同じ意味の単語でも、時制（過去形・現在形）や単数・複数、比較級などで語尾が様々に変化します。

* **例**: `go`, `went`, `going`, `goes` / `dog`, `dogs` / `better`, `good`

これらをそのまま別々の単語として扱うと、語彙数（辞書のサイズ）が膨大になり、モデルの学習が非効率になります。
そこで、これらを**共通の「基本形」に変換して統一する処理**を正規化と呼び、その代表例がステミングと見出し語化です。

---

## 2. ステミング (Stemming)

* **仕組み**: 辞書を使わず、**機械的なルール（文字列操作）**に基づいて単語の末尾を削り落とす手法です。
* **特徴**:
  * 非常に高速で処理がシンプルです。
  * 辞書を使わないため、**「実在しない単語（スペルミスのような文字列）」** に変換されることがよくあります。
* **変換例**:
  * `studying` ──> `studi` (末尾をカット)
  * `studies` ──> `studi`
  * `went` ──> `went` (ルールでカットできない不規則変化はそのまま残る)

---

## 3. 見出し語化 (Lemmatization)

* **仕組み**: **辞書（語彙データベース）** を参照し、文脈における**品詞（名詞、動詞、形容詞など）を判別した上で**、正しい「原形（見出し語）」に変換する手法です。
* **特徴**:
  * 文脈を考慮するため、常に**「実在する正しい英単語」** に変換されます。
  * 辞書引きや品詞の自動判定を伴うため、ステミングに比べて処理が重くなります。
* **変換例**:
  * `studying` / `studies` ──> `study` (正しい原形)
  * `went` ──> `go` (過去形から原形へ)
  * `better` ──> `good` (比較級から原形へ)

---

## 4. 比較まとめ

| 比較項目 | ステミング (Stemming) | 見出し語化 (Lemmatization) |
| :--- | :--- | :--- |
| **アプローチ** | 語尾を機械的に削る (ルールベース) | 辞書を引いて品詞を考えて戻す (言語学ベース) |
| **処理スピード** | 🚀 非常に速い | 🐢 やや遅い |
| **変換の正確さ** | 粗い (辞書にない綴りになる) | 正確 (常に正しい辞書の見出し語) |
| **`went` の変換** | `went` (そのまま) | `go` |
| **`better` の変換** | `better` (そのまま) | `good` |

---

## 5. 💡 現代の LLM（GPT や BERT）における位置づけ

かつての検索エンジンや古い分類モデルでは、これらは必須の前処理でしたが、**現代のLLMでは、ステミングや見出し語化はほぼ使われません。**

### なぜ使われなくなったのか？
1. **サブワードトークン化の普及**
   * 現代のモデルは単語をそのままではなく `study` + `##ing` のように細かく分解（サブワード化）して処理します。これにより、語彙数の爆発を防ぎつつ、時制や変化も自動でカバーできるようになりました。
2. **ニュアンスの消失を防ぐため**
   * 「`went`（過去に行った）」と「`go`（今行く）」は、ステミング等で統一すると、時系列のニュアンス情報が失われてしまいます。現代の強力なディープラーニングモデルは、こうした細かい語尾変化による文脈のニュアンスも理解できるため、あえて情報を消さずに生のまま入力するのが主流になっています。
