# フィードフォワードネットワーク（FFN）の役割とアテンションとの違い

書籍の111ページ（リスト4-4）に、`FeedForward` クラスが実装されています。

```python
class FeedForward(nn.Module):
    def __init__(self, cfg):
        super().__init__()
        self.layers = nn.Sequential(
            nn.Linear(cfg["emb_dim"], 4 * cfg["emb_dim"]),
            nn.GELU(),
            nn.Linear(4 * cfg["emb_dim"], cfg["emb_dim"])
        )
```

この「フィードフォワード」とはどういう意味なのか、そしてTransformerブロックにおいてアテンション層とどのような役割分担をしているのかについて解説します。

---

## 1. 「フィードフォワード（Feed-Forward）」とは？

英語の **Feed（供給する、送る）** と **Forward（前方へ）** を組み合わせた言葉で、**「データをひたすら前方（出力側）へ送り届ける」**という意味です。

### 🔄 ループ（逆戻り）を持たないネットワーク構造
フィードフォワードネットワークとは、データが一方向（入力 $\to$ 隠れ層 $\to$ 出力）にのみ流れ、途中でデータが逆戻りしたり、過去の状態を記憶して循環（ループ）したりする構造を一切持たないニューラルネットワークのことです。

*   **かつての言語モデル (RNN: 回帰型ニューラルネット)**:
    「過去の単語の記憶」を次のステップの入力に混ぜるという**ループ（フィードバック）構造**を持っていました。
*   **Transformer / GPT モデル (FFN: フィードフォワード層)**:
    過去の記憶をループさせるような複雑な構造は完全に排除され、各トークンのデータを一方向の計算だけで前へ前へと流すシンプルな構成になっています。

---

## 2. Transformerブロック内における完璧な役割分担

Transformerの1つの「ブロック（層）」の中は、大きく分けて**アテンション層**と**フィードフォワード層**の2つの主役がコンビを組んで動いています。

```text
  ┌────────────────────────────────────────────────────────┐
  │                 Transformerブロック                     │
  │                                                        │
  │   1. マルチヘッド・アテンション (MHA)                  │
  │      単語と単語の「関係性」を調べる。                  │
  │      ⇒ 【単語間で情報をブレンドする（横の統合）】       │
  │                  │                                     │
  │                  ▼                                     │
  │   2. フィードフォワード・ネットワーク (FFN)            │
  │      文脈が混ざった後、各単語が「個別に深く考える」。   │
  │      ⇒ 【単語同士を混ぜず、単体で特徴変換する（縦の抽出）】│
  └────────────────────────────────────────────────────────┘
```

### ① アテンション層 ──「情報を混ぜ合わせる（横のブレンド）」
文脈を理解するために、「どの単語が、他のどの単語に注目すべきか」を計算し、**単語間で情報を混ぜ合わせる**役割を担います。ここでは、他の単語のデータが自分のデータに混ざり合います（トークン間の相互作用）。

### ② フィードフォワード層 (FFN) ──「個別に深く考える（自己の咀嚼）」
アテンションによって他の単語の情報が程よく混ざり合った後、各単語（トークン）が**「他人のデータは一切見ず、自分自身のデータだけを見て独立して特徴を深く変換（咀嚼）する」**役割を担います。
ここでは、単語間で情報が混ざることは一切ありません（トークンごとの独立処理）。

---

## 3. なぜ次元を 4倍（`4 * emb_dim`）に広げて縮小するのか？

書籍のコードでも、中間層の次元数を元の埋め込み次元（`emb_dim`）の **4倍** に拡大しています。

$$emb\_dim \xrightarrow{\text{拡大 (Linear)}} 4 \times emb\_dim \xrightarrow{\text{活性化 (GELU)}} emb\_dim \xrightarrow{\text{縮小 (Linear)}}$$

これには、モデルの「記憶容量」を増やす決定的な意味があります。

### 🧠 「事実知識の格納庫（ストレージ）」としての役割
アテンション層が「どの言葉がどの言葉に関係しているか」という**情報の交通整理（ルーティング）**を担うのに対し、フィードフォワード層はモデルが持つ**知識の記憶（ストレージ）**を担っていると言われています。

情報を一度4倍の広い次元（高次元空間）に引き伸ばし、そこで非線形な活性化関数（GELU）を通すことで、モデルの表現力の許容量（キャパシティ）を大きく広げます。この広い空間を利用して、モデルは「Aという単語とBという単語がアテンションで混ざったら、これは歴史上の人物の話である」といった**複雑な事実関係や概念のパターンを知識としてモデルの重みの中に記憶**します。

そして、十分に深く処理したあと、再び元の次元（`emb_dim`）に圧縮して次の層へ引き渡します。
