# GELU, SwiGLU, 標準ガウス分布の数理と LLM での役割

書籍111ページのリスト4-3に登場する `GELU` 活性化関数、現代のLLM（LLaMAやGemmaなど）のデファクトスタンダードである `SwiGLU`、そしてその基盤となる確率統計の概念である「標準ガウス分布」について、図解と数式を交えて詳しく解説します。

---

## 1. 基礎概念：標準ガウス分布（標準正規分布）とは？

標準ガウス分布とは、**「平均が 0、分散（標準偏差）が 1 の、左右対称な釣鐘型の確率分布」**のことです。

```text
       確率密度 (PDF)
         │       ▲ (平均0が最も確率が高い)
         │     /   \
         │    /     \
   ──────┼───/───────\─── x (入力値)
        -2  -1   0   1   2
```

この分布には、2つの重要な数式（関数）があります。

### ① 確率密度関数（PDF: Probability Density Function）
ある入力値 $x$ が発生する「確率の高さ（グラフの縦軸の値）」を示す関数です。

### ② 累積分布関数（CDF: Cumulative Distribution Function / 記号: $\Phi(x)$）
$-\infty$ から入力値 $x$ までの「確率の合計値（グラフの左側からの面積）」を示す関数です。
*   **$x$ が非常に小さい（マイナス）とき**: 面積はほぼ `0.0` になります（$\Phi(x) \approx 0$）。
*   **$x = 0$ のとき**: ちょうど真ん中なので、確率はぴったり半分（$\Phi(x) = 0.5$）です。
*   **$x$ が非常に大きい（プラス）とき**: 面積はほぼ `1.0` になります（$\Phi(x) \approx 1$）。

---

## 2. GELU（Gaussian Error Linear Unit）とは？

GELUは、このガウス分布の累積分布関数 $\Phi(x)$ を利用した活性化関数です。

### 📈 数式
$$\text{GELU}(x) = x \cdot \Phi(x)$$

### 💡 ゲートの確率的制御（直感的な意味）
GELUの掛け算（$x \times \Phi(x)$）は、**「入力値 $x$ 自体に基づいて、その情報をどれだけの確率で後続の層へ通すか（ゲートを開くか）」**をコントロールしています。

*   **入力 $x$ が大きなプラス値（例: +3.0）のとき**:
    累積確率 $\Phi(x)$ はほぼ `1.0` です。したがって、出力は $x \times 1.0 = x$ となり、**情報はそのまま通過**します。
*   **入力 $x$ が大きなマイナス値（例: -3.0）のとき**:
    累積確率 $\Phi(x)$ はほぼ `0.0` です。したがって、出力は $x \times 0.0 = 0.0$ となり、**情報は完全に遮断**されます。
*   **入力 $x$ が $0$ 近辺のとき**:
    $\Phi(x)$ は $0.5$ 付近になります。情報は半分通され、なだらかな曲線を描きます。

### ⚙️ 近似式が使われる理由
厳密な $\Phi(x)$ の計算には、誤差関数 `erf`（積分計算）が必要で、コンピュータにとっては非常に重い処理です。そのため、計算を高速化するために `tanh`（双曲線正接）を用いた以下の**近似式**が GPT-2 や書籍の実装（リスト4-3）で使われています。

$$\text{GELU}(x) \approx 0.5 \cdot x \cdot \left( 1 + \tanh\left[ \sqrt{\frac{2}{\pi}} \cdot (x + 0.044715 \cdot x^3) \right] \right)$$

## 3. GELU が ReLU より優れている理由

### ① すべての点で滑らか（微分可能）
ReLUは $x=0$ の地点でグラフが「直角」に折れ曲がっており、そこでは数学的に微分できません。一方、GELUはすべての地点で滑らかに繋がっているため、**学習時の勾配（重みを調整するヒント）が安定し、モデルの学習が効率よく収束します。**

### ② デッドニューロン（Dead ReLU）問題の回避
ReLUは $0$ 以下の領域で傾き（微分値）が完全に `0` になります。そのため、一度学習の過程でニューロンの出力がマイナス領域に入ってしまうと、そのニューロンは勾配を失って二度とアップデートされなくなる「デッドニューロン現象」が起きます。
GELUは、グラフを見ると分かる通り、**負の領域でも $x \approx -1.0$ あたりでわずかにマイナス側に回り込んでから $0$ に戻る**ため、負の入力に対しても微小な勾配が存在し、ニューロンが完全に死ぬのを防ぎます。

## 4. SwiGLU（Swish-Gated Linear Unit）とは？

GELUの考え方をさらに発展させ、**現代の最強のLLM（LLaMA, Gemma, PaLM, Mistralなど）で標準採用されている活性化関数**です。

### ① 要素の分解：Swish（スウィッシュ）と GLU
SwiGLUを理解するために、まず2つの要素に分解します。

#### A. Swish 関数 (または SiLU)
GELUと非常に形状がよく似た滑らかな活性化関数で、シグモイド関数 $\sigma$ を用いて計算されます（PyTorchの `F.silu` と等価）。
$$\text{Swish}(x) = x \cdot \sigma(\beta x)$$

#### B. GLU (Gated Linear Unit)
入力 $x$ に対して2つの異なる線形変換（Linear層） $x W$ と $x V$ を行い、**一方をゲート（情報のフィルター）として使い、もう一方のデータに要素ごとに掛け合わせる**構造です。
$$\text{GLU}(x) = (x W + b) \otimes \sigma(x V + c)$$

### ② SwiGLU の定義
GLUのゲート制御に使うシグモイド関数 $\sigma$ を、より表現力の高い **Swish** 関数に置き換えたものが **SwiGLU** です。

$$\text{SwiGLU}(x) = (x W + b) \otimes \text{Swish}(x V + c)$$

### 🏆 なぜ SwiGLU は強力なのか？
GELUは、入力 $x$ の値そのものからゲートの開き具合を直接計算していました。
一方、SwiGLUは、入力されたデータを一度2つの異なるLinear層（$W$ と $V$）に分岐させ、**「流すデータそのもの（Value）」**と**「各要素のゲートの開き具合（Gating）」**を別々のパラメータで学習させます。

これにより、モデルは「どの情報を通過させ、どの情報を遮断するか」の判断ルールをより柔軟に、高い表現力で学習できるようになります。この表現力の高さが、最新LLMの圧倒的な性能を支えています。
