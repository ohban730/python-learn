# LayerNorm における scale と shift 変数の役割

書籍の108ページ（リスト4-2）の `LayerNorm` クラス内に、以下のコードがいきなり登場します。

```python
self.scale = nn.Parameter(torch.ones(emb_dim))
self.shift = nn.Parameter(torch.zeros(emb_dim))
```

これらの変数が何者であり、なぜ LayerNorm にとって不可欠なのかについて、その本質を解説します。

---

## 1. `scale` と `shift` とは何か？

一言で言うと、**「標準化（平均0・分散1に揃える処理）の後に、モデルが学習を通じて、データを最適な平均と分散に再調整（拡大縮小・平行移動）できるようにするためのパラメータ」**です。

数学や教科書・論文の数式では、それぞれ以下のように表記されます。
*   `self.scale` $\to$ **$\gamma$ (ガンマ)**
*   `self.shift` $\to$ **$\beta$ (ベータ)**

LayerNorm全体の計算式は、入力 $x$ に対し、平均を $\mu$、分散を $\sigma^2$ とすると、以下のようになります。

$$y = \text{scale} \times \left( \frac{x - \mu}{\sqrt{\sigma^2 + \epsilon}} \right) + \text{shift}$$

---

## 2. なぜ「平均0・分散1」に固定してはダメなのか？

LayerNormの目的は「データを綺麗に標準化（平均0・分散1）して、学習の不安定さを解消すること」です。しかし、**平均0・分散1というデータ分布が、常にネットワークの表現力にとって最適であるとは限りません。**

これらが必要な理由は主に2つあります。

### ① 活性化関数（ReLUやGELUなど）との衝突を避けるため
平均を `0`、分散を `1` に綺麗に揃えると、**データの約半分（$50\%$）はマイナスの値**になります。
しかし、この層の直後には活性化関数（ReLUなど）が控えています。ReLUは「マイナスの入力をすべて $0$ にカットする」関数です。

もしデータを平均0・分散1に完全に固定してしまうと、**せっかく綺麗に正規化した情報の約半分が、活性化関数を通った瞬間に強制的に消滅（失活）してしまいます。**

```text
■ もし scale と shift が無い場合：
  データ(平均0) ──> 活性化関数(ReLU) ──> 情報の半分が 0 になって消滅！ (表現力の低下)

■ scale と shift がある場合：
  学習によって shift (平均) をプラス方向にずらすことで、
  「活性化関数を通しても情報が消えないように、データを安全なプラス領域へ逃がす」
  といった調整をモデル自身が自律的に行えるようになります。
```

### ② 表現力の復元
正規化（標準化）は、各ニューロンに入力される値の範囲を強制的に固定してしまうため、いわば「モデルの個性を消す」操作です。
最後に `scale`（掛け算による拡大縮小）と `shift`（足し算による平行移動）を適用することで、モデルは**「学習に都合の良い綺麗さを保ちつつ、必要に応じてデータの個性（分布の広がりや偏り）を復元する」**ことができます。

---

## 3. なぜ `nn.Parameter` として定義されているのか？

`scale` と `shift` は、PyTorchの **`nn.Parameter`** という特殊なクラスで包まれています。

```python
self.scale = nn.Parameter(torch.ones(emb_dim))
```

これにより、PyTorchはこれらの変数を「学習によって更新すべき重み（パラメータ）」として自動認識します。
*   モデルの他の重み（Linear層のバイアスや重みなど）と同様に、学習プロセスで「勾配」が計算され、**モデルが自分にとって最も都合の良いスケール（$\gamma$）とシフト（$\beta$）の値を自律的に学習していきます。**

---

## 4. なぜ初期値が `ones` (1) と `zeros` (0) なのか？

*   `scale` の初期値: `torch.ones(emb_dim)` $\to$ すべて **`1.0`**
*   `shift` の初期値: `torch.zeros(emb_dim)` $\to$ すべて **`0.0`**

これは、**「学習のスタート時点では、LayerNormは純粋な標準化（平均0・分散1）を行う状態から始めたいから」**です。

初期値をあてはめると、最後の計算は以下のようになります。

$$y = 1.0 \times \text{norm\_x} + 0.0 = \text{norm\_x}$$

このように、初期状態では何も変化を加えずに標準化されたデータをそのまま流し、そこから徐々に学習を進めることで、モデルが必要に応じて最適な形状へと変形させていきます。
