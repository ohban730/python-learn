# CNN、RNN の違い、および ResNet の解説

深層学習（ディープラーニング）の基礎となる2つの代表的なモデル構造（CNN / RNN）の違いと、画像認識の歴史を塗り替え、現在のTransformerにも受け継がれている重要技術 **ResNet** について解説します。

---

## 1. 畳み込みニューラルネットワーク (CNN) とは？

**CNN (Convolutional Neural Network)** は、主に**画像認識やコンピュータビジョン**の分野で圧倒的な成果を上げてきたネットワーク構造です。

```
[ 入力画像 ] ──> [ 畳み込み層 (特徴抽出) ] ──> [ プーリング層 (圧縮) ] ──> [ 全結合層 ] ──> [ 判定結果 ]
```

### 主な特徴と仕組み
* **畳み込み層 (Convolutional Layer)**
  * 画像全体に対して「フィルタ（カーネル）」をスライドさせながら掛け算を行い、画像から「縦の線」「横の線」「輪郭」などの局所的な特徴を抽出します。
* **プーリング層 (Pooling Layer)**
  * 特徴マップを縮小（一般的にはエリア内の最大値をとるMax Poolingなど）し、物体の「位置が少しズレても同じものとみなす」頑健性を持たせます。
* **空間情報の保持**
  * 画像の縦・横ピクセルの位置関係（空間情報）を崩さずに処理することができます。また、フィルタを画像全体で共有（重み共有）するため、パラメータ数を劇的に削減できます。

---

## 2. CNN と RNN の違い

**RNN (Recurrent Neural Network: 再帰型ニューラルネットワーク)** は、CNNとは対照的に**時系列データ（文章、音声、時間変化するデータ）**の処理に特化した構造です。

両者の決定的な違いは以下の通りです。

| 比較項目 | CNN (畳み込みニューラルネットワーク) | RNN (再帰型ニューラルネットワーク) |
| :--- | :--- | :--- |
| **得意なデータ** | **空間的データ**（画像、動画の1フレームなど） | **時間的・シーケンスデータ**（文章、音声、株価など） |
| **データの関連性** | 縦・横の「隣り合うピクセル」に強い関連がある | 「前後の並び順（時間系列）」に強い関連がある |
| **情報の処理方法** | 全体を一気に並列処理する（前向き伝播） | データを1ステップ（1単語等）ずつ順に処理する |
| **内部メモリ（状態）** | 保持しない（入力に対して常に同じ出力を出す） | 保持する（過去の情報を「隠れ状態」として次に引き継ぐ） |
| **接続の構造** | 一方向のフィードフォワード（順方向のみ） | ループ（再帰）構造を持ち、過去の出力を自身に入力する |

* **直感的なイメージ**:
  * **CNN** は「写真全体をパッと見て特徴を見つけるカメラ」
  * **RNN** は「小説を左から右へ、前の文脈を頭に留めながら読み進める読書家」

---

## 3. ResNet (Residual Network) とは？

**ResNet** は、2015年に Kaiming He 氏らによって提案され、画像認識コンペティション（ILSVRC）で圧倒的な成績を収めた革新的なCNNのアーキテクチャです。

### ① ResNet が解決した課題
深層学習では「層を深く（ディープに）するほど、より複雑な特徴が学べて精度が上がる」と考えられていました。
しかし、単純に層を50層、100層と深くしていくと、以下の問題が発生しました。
* **勾配消失問題**: 逆伝播のときに勾配が途中で消えてしまい、浅い層が学習できなくなる。
* **劣化問題 (Degradation Problem)**: 訓練データに対する精度すら、浅いモデルより悪くなってしまう。

### ② 解決策：残差接続（Skip Connection / Residual Connection）
ResNetは、層を飛び越えて（スキップして）入力を後ろの層に直接足し合わせる **「残差接続（スキップ接続）」** を導入しました。

```
               x (入力)
               │ ──┐
               ▼   │
        [ 畳み込み層 ] │ (ショートカット / スキップ)
               ▼   │
        [ 畳み込み層 ] │
               ▼   │
        [加算器 (+)] <─┘
               │
             F(x) + x  (出力)
```

通常のネットワークは「入力 $x$ から出力 $H(x)$ を直接予測する」ように学習しますが、ResNetは「入力 $x$ と出力の差分（残差）である $F(x) = H(x) - x$」を予測するように学習します。

### ③ なぜこれで深くできるのか？
1. **勾配のバイパス（抜け道）ができる**
   * 逆伝播の際、加算器（+）を通ることで、勾配が「畳み込み層を通るルート」と「スキップ接続を通るルート」に分岐します。スキップ接続側は重みの掛け算を一切しないため、**勾配が減衰せずにそのまま（勾配 $1$ で）一気に最初の層まで届きます**。これにより勾配消失が完全に防がれます。
2. **恒等写像（何もしないこと）の学習が容易**
   * もしその層が不要だった場合、通常の層は「入力をそのまま出力する（恒等写像 $H(x) = x$）」という複雑なパラメータ調整が必要ですが、ResNetなら単に畳み込み層の重みをゼロ（$F(x) = 0$）にするだけで、自動的に $H(x) = x$ になります。

---

## 4. LLM（Transformer）との関連性

実は、この **残差接続（Skip Connection）** は、現代のGPTやBERTなどの **Transformer** の中でも非常に重要な役割を果たしています。

Transformerの各ブロック（Self-Attention層やFFN層）の前後には、すべてこの「残差接続」が組み込まれており、これがあるからこそ、何十層、何百層ものレイヤーを持つ超巨大なLLMでも勾配消失を起こさずに安定して事前学習を成功させることができます。
