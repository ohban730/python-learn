# PyTorch の `F.one_hot` で `num_classes` を小さくするとどうなるか？

PyTorch の関数 `F.one_hot(input_ids, num_classes=...)` において、`num_classes`（クラス数＝ベクトルの次元数）の数値を小さく設定した場合の挙動とエラーについて解説します。

---

## 1. 結論：入力IDの最大値以下にすると、エラーでクラッシュします

`num_classes` に指定した値が、`input_ids` の中に入っている**「最大のID値」以下**になると、PyTorch は以下のエラー（RuntimeError）を発生させて処理を停止します。

```text
RuntimeError: Class values must be smaller than num_classes.
(クラスの値は num_classes よりも小さくなければなりません)
```

---

## 2. なぜエラー（クラッシュ）が起きるのか？

one-hot化は、指定されたインデックス（IDの位置）だけを `1` にし、他を `0` にする処理です。

例えば、`input_ids` にID `4`（例えば5番目の文字）が含まれているとします。このとき `num_classes=3`（3次元ベクトル）に設定したとします。

```text
【 num_classes = 3 のベクトル 】
  [0, 0, 0]  (インデックスは 0, 1, 2 のみ)

【 そこに ID: 4 を one-hot化しようとすると... 】
  ベクトルの 4番目（インデックス4）の位置を 1 にしたいが、
  サイズが3しかないので、アクセスする場所が存在しない（インデックス範囲外エラー）。
```

このように、ベクトルの箱のサイズ（`num_classes`）を超えたインデックスを指定することになるため、プログラムが強制終了します。

---

## 3. 入力内の最大値よりは大きいが、全体の語彙数より小さくした場合は？

例えば、全体の語彙数（`len(token2idx)`）は `20` だが、入力されたテキストの中にたまたま `10` 以上のIDが登場せず、`num_classes=11` と小さく設定した場合です。

* **挙動**: エラーにはならず、Shapeが `(文字数, 11)` の2次元テンソルが正常に出力されます。
* **発生する問題**:
  * 一見動きますが、後続のニューラルネットワークのレイヤー（例: Linear層やEmbedding層）は、入力ベクトルの次元数が **`20` であることを前提**に設計されています。
  * そのため、その後にネットワークにこの `(文字数, 11)` のテンソルを入力した時点で、**「次元が合いません（Dimension mismatch）」** という別エラーが発生し、結局処理は失敗します。

---

## 4. `num_classes` を指定しない（省略する）とどうなる？

`F.one_hot(input_ids)` と引数を省略した場合、PyTorch は自動的に **「入力（`input_ids`）の中の一番大きい数値 ＋ 1」** を `num_classes` として自動計算します。

* **問題点**: 
  * 入力するテキストごとに、出現する最大IDの値は変わります（短い文なら最大IDが `5`、長い文なら最大IDが `19` など）。
  * そのため、テキストごとに生成される one-hot ベクトルの長さ（次元数）がバラバラになってしまい、やはりニューラルネットワークに入力できなくなります。

### 鉄則
ニューラルネットワークに入力するベクトルサイズは、常に固定されている必要があります。そのため、常に **`num_classes=len(token2idx)`（語彙数全体）** を明示的に指定して、ベクトルの長さを固定するのが鉄則です。
