# パープレキシティ (Perplexity) の数式と数理モデル

書籍147ページにおいて、モデルの性能を解釈するための指標として **「パープレキシティ (Perplexity / PPL)」** が登場します。

「なぜ、損失 (Loss) の指数（$e^{\text{Loss}}$）を計算するだけで『モデルが実質的に何個の単語で迷っているか』という具体的な選択肢の数に変換されるのか」、その数学的証明と数理的な意味について詳しく解説します。

---

## 1. パープレキシティの数学的定義

情報理論におけるパープレキシティの定義は、交差エントロピー $H(p, q)$ を用いて以下のように表されます。

$$PPL = b^{H(p, q)}$$

ここで $b$ は対数の底（ベース）です。

PyTorchなどの深層学習フレームワークにおける交差エントロピー損失 $L$ は、自然対数 $e$（ネイピア数 $\approx 2.718$）を底として計算されます。そのため、LLM開発におけるパープレキシティは以下のシンプルな数式で計算されます。

$$PPL = e^L$$

```python
# PyTorchでの計算コード
perplexity = torch.exp(loss)
```

---

## 2. なぜ「実質的な選択肢の個数」になるのか？ (数理的証明)

パープレキシティが「モデルが次の単語を予測する際、平均して何個の単語（選択肢）の間で迷っているか（分岐係数: Branching Factor）」と解釈できることを、3つのケースを想定して数学的に証明します。

### 📋 前提
モデルの語彙サイズを $V$（GPT-2なら $50,257$）とします。

---

### ケース A：完全に迷っている状態（ランダム予測）
モデルが全く学習しておらず、どの単語が次に来るかさっぱりわからないため、すべての単語に均等に確率を割り振った（あてずっぽうな）状態を考えます。

このとき、正解単語に対する予測確率 $p_{\text{target}}$ は、語彙全体の等確率になるため以下のようになります。
$$p_{\text{target}} = \frac{1}{V}$$

このときの交差エントロピー損失 $L$ を計算します。
$$L = - \ln(p_{\text{target}}) = - \ln\left(\frac{1}{V}\right) = \ln(V)$$

この Loss の指数を取ってパープレキシティ $PPL$ を計算します。
$$PPL = e^L = e^{\ln(V)} = V$$

#### 💡 結論
あてずっぽうの時、パープレキシティは語彙サイズ $V$ と完全に一致します。
これは、**「モデルは語彙に含まれる $V$ 個の単語すべてにおいて迷っている（確信度ゼロ）」** という直感と数学的に100%一致します。
*   GPT-2（語彙数 50,257）であれば、初期状態の PPL は **`50257.0`** になります。

---

### ケース B：完璧に予測できている状態（迷いゼロ）
モデルが完全に学習し、正解単語を 100% の自信で言い当てられる（正解の確率が $1.0$）状態を考えます。

$$p_{\text{target}} = 1.0$$

このときの交差エントロピー損失 $L$ を計算します。
$$L = - \ln(1.0) = 0$$

このときのパープレキシティ $PPL$ を計算します。
$$PPL = e^0 = 1$$

#### 💡 結論
完璧な予測の時、パープレキシティは **`1`** になります。
これは、**「モデルが迷っている選択肢の数は 1 個（＝正解の1択に完全に絞り込めており、迷いが全くない）」** ことを意味します。

---

### ケース C：N 個の選択肢で均等に迷っている状態（N択問題）
モデルが「候補となる $N$ 個の単語」の間で均等に迷っており、それぞれの単語の予測確率が $1/N$ である状態を考えます（他の無関係な単語の確率は $0$）。

正解単語もこの $N$ 個の候補に含まれるため、その確率は以下のようになります。
$$p_{\text{target}} = \frac{1}{N}$$

このときの交差エントロピー損失 $L$ は以下のようになります。
$$L = - \ln\left(\frac{1}{N}\right) = \ln(N)$$

このときのパープレキシティ $PPL$ を計算します。
$$PPL = e^L = e^{\ln(N)} = N$$

#### 💡 結論
パープレキシティは、モデルが**「実質的に $N$ 個の単語の間で迷っている」**という分岐状態の数と、完全に等しくなります。

## 3. なぜ Loss の数値よりも評価指標として好まれるのか？

自然言語処理の研究や実プロダクトにおいて、Lossの数値そのものよりもパープレキシティが好まれる最大の理由は、**「人間の直感で性能を測定しやすいから」**です。

モデルAとモデルBを比較した時、以下のような性能評価が得られたとします。

*   **モデル A**: $\text{Loss} = 3.5$
*   **モデル B**: $\text{Loss} = 4.2$

この Loss の差「0.7」がどれくらい凄いのか、人間にはパッと見でピンときません。
しかし、これをパープレキシティ（$e^{\text{Loss}}$）に直してみます。

*   **モデル A の PPL**: $e^{3.5} \approx 33.1$ （平均 **33個** の単語で迷っている）
*   **モデル B の PPL**: $e^{4.2} \approx 66.7$ （平均 **66個** の単語で迷っている）

このように変換することで、
「モデルAは、モデルBに比べて**『迷う単語の選択肢を半分（66個 ─> 33個）にまで絞り込めている』**ので、圧倒的に優秀である」
ということが、数字のスケール感として非常に分かりやすくなります。

## 4. パープレキシティ誕生の歴史的背景：IBMの音声認識

パープレキシティの起源は、ディープラーニング以前の **1970年代の音声認識の黎明期** に遡ります。

1977年、米IBMのワトソン研究所で、世界初の本格的な統計的音声認識システム（Tangoraプロジェクト）の開発が進められていました。耳で聞こえた音（例: 「ライト」）が、`write`（書く）なのか `right`（正しい）なのかを特定するために、文脈からふさわしい単語を予測する「言語モデル」の性能測定が必要になりました。

当時、モデルの性能は情報理論の数式通りに「エントロピー（単位: ビット）」で表されていましたが、これでは音声学のエンジニアたちにとって「どれくらい予測が楽になったのか」がピンときませんでした。

そこで、プロジェクトのメンバーであったフレッド・ジェリネック（Fred Jelinek）らは、エントロピーを逆算（指数化）し、**「モデルが平均して何個の単語の間で迷っている（当惑している）か」** を表す指標を考案しました。
英語の **Perplexity（当惑度、複雑さ）** という言葉がそのまま指標の名前になり、言語モデルの標準的な評価基準として定着しました。

ちなみにシャノンらの研究によると、英語の普通の文章を読むとき、**人間のパープレキシティは「10〜20」程度**（＝次に何が来るかを、平均10〜20択程度まで無意識に絞り込めている）と見積もられています。

## 5. コラム：物理（熱力学）のエントロピーと情報エントロピーの繋がり

「エントロピー」と聞くと、物理の熱力学で習う「分子の乱雑さ（無秩序さ）」をイメージする方が多いかもしれません。
実は、**物理のエントロピーと、情報理論のエントロピーは、数学的に全く同じ構造（数式）を持っています。**

この2つを繋ぐ、コンピュータ科学の歴史における有名な逸話があります。

1948年にクロード・シャノンが情報の「予測のしづらさ（不確実性）」を表す数式を導き出したとき、これを何と命名すべきか迷っていました。そこでシャノンは、天才数学者の **ジョン・フォン・ノイマン** に相談しました。ノイマンはこう答えました。

> 「それを**『エントロピー』**と呼ぶといい。理由は2つある。
> 1つ目に、この数式は統計力学におけるエントロピーの式と全く同じ形をしているからだ。
> 2つ目に、そしてこれが最も重要な理由だが、**誰もエントロピーが本当は何なのかをよく理解していない。だから、議論で君が常に有利になれるからだ。**」

このノイマンの冗談混じりの提案がきっかけで、「情報エントロピー」と命名されました。

### 📊 2つのエントロピーの共通の本質

物理のエントロピーと情報のエントロピーは、どちらも本質的に **「状態の散らばりの多さ（不確実性）」** を表しています。

*   **物理のエントロピー (熱力学)**:
    コップの中の熱や気体分子が、どれくらい秩序なく**「バラバラに散らばっているか」**（乱雑さ）。
*   **情報のエントロピー (情報理論)**:
    次に来る単語や文字が、どれくらい偏りなく**「バラバラに散らばっているか」**（予測のしづらさ）。
    *   特定の1単語だけが 100% の確率で来るなら、散らばりは 0 なのでエントロピーは最小（0）。
    *   全50,257単語がまったく等しい確率で散らばっているなら、エントロピーは最大。

LLMの学習とは、この「情報の散らばり（情報エントロピー）」をどんどん低くしていき、次の単語をピンポイントで予測できるよう「不確実性をそぎ落としていく」プロセスそのものなのです。
