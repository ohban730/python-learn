# ビーム探索（Beam Search）の仕組みと特徴

テキスト生成AIやLLMが文章を出力する際、次の単語をどう選択していくかという「探索アルゴリズム」の一つである **ビーム探索（Beam Search）** について解説します。

---

## 1. なぜビーム探索が必要なのか？（貪欲法の限界）

テキストを生成する最もシンプルな方法は **貪欲法（Greedy Search）** です。これは、「その瞬間に最も確率が高い単語を1つだけ選び、それを繋げていく」方法です。

* **貪欲法の問題点**: 
  * 「一歩先」しか見ていないため、最初に少しでも微妙な単語を選んでしまうと、その後の文章全体のバランスが崩れ、修復できなくなります（局所最適な選択が、全体最適な選択になるとは限らない）。

---

## 2. ビーム探索の仕組み

ビーム探索は、貪欲法のように「1つのルートだけを信じて進む」のではなく、**「常に上位N個（ビーム幅: `num_beams`）の生き残り候補（ルート）をキープしながら、並行して文章を伸ばしていく」**手法です。

### 動作イメージ（ビーム幅 = 2 の場合）

```
【ステップ 1】最初の単語候補
  * 候補A: 「私は」 (確率: 60%)  ──> ★キープ
  * 候補B: 「昨日」 (確率: 30%)  ──> ★キープ
  * 候補C: 「犬が」 (確率: 10%)  ──> 脱落

【ステップ 2】2つ目の単語候補（それぞれのルートから分岐）
  * ルートAから派生:
    * 「私は」 -> 「猫を」 (累積確率: 45%)  ──> ★キープ
    * 「私は」 -> 「犬を」 (累積確率: 10%)  ──> 脱落
  * ルートBから派生:
    * 「昨日」 -> 「私は」 (累積確率: 25%)  ──> ★キープ
    * 「昨日」 -> 「学校へ」(累積確率: 5%)   ──> 脱落

【ステップ 3】以降も同様に、常に累積確率が高い上位2つの「ルート」だけをキープしながら単語を伸ばしていきます。

最終的に文章が完成した時点で、最も確率の重みが高かった1つのルートを答えとして出力します。
```

---

## 3. ビーム探索のメリット・デメリット

| メリット | デメリット |
| :--- | :--- |
| ⭕ **高品質で論理的**: 全体として最も自然で確率が高い文章を見つけ出せる。 | ❌ **計算コストが高い**: ビーム幅（`num_beams`）の数だけ並列で計算するため、メモリと処理時間が増える。 |
| ⭕ **翻訳や要約で強力**: 一言一句の正確性が求められるタスクに非常に強い。 | ❌ **文章が単調・退屈になる**: 常に安全で確率の高い単語を選ぶため、小説の執筆やフリートークなどのクリエイティブなタスクでは、お決まりのつまらない表現になりがち。 |

---

## 4. 他の生成手法との使い分け

現代のLLMでは、解くべきタスクに合わせて探索方法を切り替えています。

### ① 決定論的なタスク（正解があるもの）
* **手法**: ビーム探索（`num_beams=4` など、`do_sample=False`）
* **向いているタスク**: 翻訳、文章の要約、ソースコードの生成、計算問題
* **理由**: 創造性は不要であり、最も確実で論理的に正しいテキストを出力したいため。

### ② 創造的なタスク（正解が一つではないもの）
* **手法**: サンプリング（`do_sample=True`、`temperature=0.7`, `top_p=0.9` など）
* **向いているタスク**: チャットボット（雑談）、小説や詩の執筆、アイデア出し
* **理由**: 毎回同じ回答になるのを防ぎ、人間味のある、ユーモアや創造性に富んだ文章を出力したいため。
