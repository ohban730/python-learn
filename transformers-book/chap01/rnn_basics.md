# リカレントニューラルネットワーク (RNN) の基礎

リカレントニューラルネットワーク（RNN: Recurrent Neural Network）は、テキストや音声などの**時系列データ（系列データ）**を処理するためのニューラルネットワークです。

「リカレント（再帰・循環）」という名の通り、内部に**ループ構造**を持ち、過去の情報を記憶しながらデータを先頭から順番に処理します。

---

## 1. RNN の基本構造と数式

RNNは、各時刻（ステップ） $t$ において、**現在の入力 $x_t$** と **1つ前のステップの記憶（隠れ状態） $h_{t-1}$** の両方を受け取り、現在の新しい記憶 $h_t$ を更新します。

```text
  x_t (現在の入力)
   │
   ▼
[ RNN セル ] <─── h_{t-1} (1つ前の記憶)
   │
   ├─► h_t (新しい記憶 / 次のセルへ)
   ▼
  y_t (現在の出力)
```

### 🔢 隠れ状態の更新式 (数学的表現)
時刻 $t$ における新しい隠れ状態 $h_t$ は、以下の式で計算されます。

$$h_t = \tanh(W_{ih} x_t + b_{ih} + W_{hh} h_{t-1} + b_{hh})$$

*   $x_t$ : 時刻 $t$ の入力（例: 文の中の $t$ 番目の単語ベクトル）
*   $h_{t-1}$ : 時刻 $t-1$ の隠れ状態（過去の文脈を凝縮した記憶ベクトル）
*   $W_{ih}, b_{ih}$ : 入力 $x$ から隠れ状態 $h$ へ変換するための重みとバイアス
*   $W_{hh}, b_{hh}$ : 前の隠れ状態 $h_{t-1}$ から次の隠れ状態 $h$ へ変換するための重みとバイアス
*   $\tanh$ (双曲線正接関数) : 活性化関数（出力を -1.0 〜 1.0 の範囲に収める）

このループをシーケンス（文など）の長さ分だけ繰り返すことで、文全体の意味を $h$ に蓄積していきます。

---

## 2. PyTorch での nn.RNN の仕様

PyTorchでは、`torch.nn.RNN` クラスを使用して簡単にRNNを構築できます。

### ⚙️ 主要パラメータ
`nn.RNN(input_size, hidden_size, num_layers=1, batch_first=True)`
*   `input_size` : 入力ベクトルの次元数（例: 各単語の分散表現ベクトルの次元数）
*   `hidden_size` : 隠れ状態 $h$ の次元数（記憶のキャパシティ）
*   `batch_first=True` : 入出力テンソルのShapeを `(Batch, Sequence, Features)` にする設定（推奨）

### 📐 入出力テンソルの Shape

*   **入力 (Input)**: `(Batch, Sequence_Length, Input_Size)`
    *   例: 10枚の画像や10個の文（`Batch=10`）、それぞれ 5単語（`Sequence_Length=5`）、各単語が 100次元ベクトル（`Input_Size=100`）
*   **出力 (Output)**: `(Batch, Sequence_Length, Hidden_Size)`
    *   各ステップ（時刻 $t$）でのすべての隠れ状態 $h_t$ のリスト。
*   **最終状態 (hn)**: `(Num_Layers, Batch, Hidden_Size)`
    *   シーケンスの最後（終端）まで処理し終えた、最終的な記憶 $h_T$。文章全体の感情分析などを行う場合は、この `hn` を使って分類します。

---

## 3. RNN はエンコーダー専用なのか？（エンコーダーとデコーダーの使い分け）

`torch.nn.RNN` 自体は「エンコーダー専用のモジュール」ではありません。
これは単に「系列データを再帰的に処理する基本セル」を定義するモジュールです。そのため、**「どのようにデータを流すか」**という実装方法によって、エンコーダーとしてもデコーダーとしても機能します。

### A. エンコーダーとしての役割
入力された系列（例: 「This is a pen」）を一気に読み込み、文の意味を凝縮した**「隠れ状態ベクトル（コンテキスト）」**を出力します。

*   **動作**: 入力データを一括で `rnn(x)` に通し、最終時刻の隠れ状態 `hn` を取り出します。

### B. デコーダーとしての役割
エンコーダーが作成したコンテキスト（隠れ状態）を初期値として受け取り、**「1語ずつ順番に次の言葉を生成」**します。

*   **動作**: データを一括で処理するのではなく、Pythonの `for` ループなどを用いて、**1時刻ずつ** `rnn(current_input, prev_hidden)` を呼び出します。「自分が直前のステップで出力した単語」を次のステップの入力にすることで、文章を生成していきます。

### 🎭 Seq2Seq モデルでの実装イメージ
機械翻訳など（Seq2Seq）では、同じ `nn.RNN` クラスのインスタンスを2つ（エンコーダー用、デコーダー用）作成して組み合わせて使います。

```python
class Seq2Seq(nn.Module):
    def __init__(self, vocab_size, embedding_dim, hidden_size):
        super().__init__()
        # 1. 入力を読み込むためのRNN（エンコーダー）
        self.encoder_rnn = nn.RNN(embedding_dim, hidden_size, batch_first=True)
        # 2. 出力を生成するためのRNN（デコーダー）※重みは別個に学習される
        self.decoder_rnn = nn.RNN(embedding_dim, hidden_size, batch_first=True)
```
