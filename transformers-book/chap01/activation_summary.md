# 活性化関数の使い分けと特徴まとめ

ニューラルネットワークにおける活性化関数（Activation Function）は、モデルに「非線形性」を与え、複雑なパターンを学習できるようにするためのものです。
時代とともに進化しており、**「解きたい課題（勾配消失、学習速度、表現力）」に応じて使い分けられます。**

---

## 1. 主要な活性化関数の使い分け一覧

| 活性化関数 | 数式 / イメージ | 主な使いどころ | 使い分けされる理由・メリット | デメリット / 弱点 |
|---|---|---|---|---|
| **Sigmoid** | $1 / (1 + e^{-x})$ | 二値分類の出力層、<br>LSTM/GRUのゲート | 出力が $[0, 1]$ なので、**「確率」や「情報の通過率（0%〜100%）」**を表現するのに最適。 | 中間層に使うと勾配が消失する。出力の平均が 0 でないため学習が遅い。 |
| **tanh** | $(e^x - e^{-x}) / (e^x + e^{-x})$ | **RNN/LSTM/GRUの<br>隠れ状態（記憶）の更新** | 出力が $[-1, 1]$。値の爆発を防ぎつつ、**「情報の減算・補正（マイナス値）」**を表現できる。原点対称。 | 深いネットワーク（CNNやTransformer）の中間層では、やはり勾配消失が起きる。 |
| **ReLU** | $\max(0, x)$ | **CNN（画像処理）**、<br>一般的なMLPの中間層 | 計算が**圧倒的に高速**（最大値を取るだけで指数計算がない）。入力が正なら傾きが常に $1$ なので**勾配消失が起きにくい**。 | 入力が負になると出力・傾きが常に $0$ になり、一部のニューロンが完全に学習を停止する（**Dying ReLU問題**）。 |
| **GELU** | $x \cdot \Phi(x)$<br>(※$\Phi$は正規分布の累積確率) | **Transformer (BERT, GPTなど)**<br>の中間層(FFN) | **「滑らかなReLU」**。入力が負であっても、0付近なら確率的に情報を少しだけ通過させ、ニューロンの死滅を防ぐ。表現力が非常に高い。 | 誤差関数や指数計算を含むため、ReLUに比べて計算コストが少し高い。 |
| **SwiGLU** | SwishとGLUの組み合わせ | **最新のLLM (LLaMA, PaLMなど)**<br>の中間層(FFN) | 現代のLLMで主流のゲート付き活性化関数。GELUを上回る表現力と**学習の収束の早さ**を持つ。 | パラメータ数と計算量がやや増える。 |

---

## 2. なぜ Transformer/LLM では GELU や SwiGLU なのか？

*   **「硬い」ReLU から「柔らかい」GELU へ**
    *   ReLU は $x=0$ を境に急激に折れ曲がります。また、マイナスの入力に対してバッサリと $0$ を出力するため、巨大な Transformer モデルでは多くのニューロンが機能しなくなる（死んでしまう）問題がありました。
    *   GELU は、**「入力値が小さくなるほど、情報が遮断される確率が高くなる」**という物理的・確率的な解釈を滑らかな曲線で表したものです。これにより、巨大なモデルでも勾配が滑らかに伝わり、学習が安定します。
*   **SwiGLU へのシフト**
    *   近年のLLM（LLaMAなど）では、GELUよりさらに表現力が高い **SwiGLU**（Swish関数とGated Linear Unitというゲート機構を組み合わせたもの）がデファクトスタンダードとして定着しています。

---

## 3. その他の代表的な活性化関数

### ① Leaky ReLU
*   **数式**: $\max(0.01x, x)$ （負の領域でわずかに傾きを設ける）
*   **使いどころ**: GAN（敵対的生成ネットワーク）や、Dying ReLUを回避したい画像処理タスク。
*   **理由**: 入力がマイナスになっても、完全に $0$ にせずかすかに勾配（0.01倍）を通し続けることで、ニューロンが死ぬのを防ぎます。

> [!NOTE]
> **GAN (Generative Adversarial Network: 敵対的生成ネットワーク)** とは、実在しない非常にリアルなデータ（画像や音楽）を生成するAI技術です。データを生成する「生成器 (Generator)」と、それが本物か偽物かを見分ける「識別器 (Discriminator)」の2つのニューラルネットワークを競い合わせる（敵対させる）ことで、互いの精度を極限まで高めていきます。


### ② ELU (Exponential Linear Unit)
*   **数式**: $x \text{ if } x > 0 \text{ else } \alpha(e^x - 1)$
*   **使いどころ**: 一部のディープな画像認識モデル。
*   **理由**: マイナス領域を指数関数で滑らかに表現し、平均出力値をゼロに近づけつつ、ノイズに強くします。

### ③ Softmax
*   **数式**: $\frac{e^{x_i}}{\sum e^{x_j}}$
*   **使いどころ**: **多クラス分類の出力層**（0〜9の数字分類、数万語の単語予測など）。
*   **理由**: 出力されるすべての値の合計が **$1.0$ (100%)** になるように正規化し、各クラスの「確率分布」を作ります。
