# RNNの入出力長と、RNNとTransformerの決定的な違い

系列データを扱うニューラルネットワークを学習する上で、混同しやすい「入力と出力の長さの関係」、および「RNN と Transformer（トランスフォーマー）の関係」について整理します。

---

## 1. RNN は入力文字数と同数の文字数を出力するとは限らない？

結論から言うと、**「同数とは限りません。タスク（解きたい問題）と設計によって、出力の長さは自在に変えられます」**。

RNN を使った設計パターンは、主に以下の3つに分類されます。

### ① Many-to-One (多くの入力に対して、1つの出力)
*   **特徴**: 長さがバラバラの文章を読み込み、最終的に1つの判定を出力します。
*   **タスク例**: **感情分析**（文のポジティブ/ネガティブ判定）、テキストのジャンル分類。
*   **実装**: RNNの全ステップの出力のうち、最後のステップの隠れ状態 `hn` だけを取り出して全結合層に入力します。

```text
入力: [ This ] -> [ is ] -> [ great ] (3単語)
出力:                                  -> [ ポジティブ ] (1つの出力)
```

### ② Many-to-Many (同数出力)
*   **特徴**: 入力されたトークン（文字や単語）の数と、出力されるラベルの数が完全に一致します。
*   **タスク例**: **品詞タグ付け**（各単語が「名詞」「動詞」のどれか判定）、名前付きエンティティ認識（NER）。
*   **実装**: 各ステップの出力 `output` のすべてに対して、全結合層を適用してラベルを予測します。

```text
入力: [ He ] -> [ runs ] (2単語)
出力: [ 名詞 ] -> [ 動詞 ] (2つの出力)
```

### ③ Many-to-Many (異なる長さの出力 / Seq2Seq)
*   **特徴**: 入力した長さと、出力する長さが異なります。
*   **タスク例**: **機械翻訳**（英語4単語 $\to$ 日本語3単語）、要約、対話システム。
*   **実装**: 入力を読み込む「エンコーダー用RNN」と、出力を1語ずつ紡ぎ出す「デコーダー用RNN」を組み合わせます。デコーダーは終端トークン（`<eos>` など）が出るまでループで出力し続けるため、入出力の長さが一致しなくなります。

```text
入力（英語）: [ This ] -> [ is ] -> [ a ] -> [ pen ] (4単語)
出力（日本語）: [ これ ] -> [ は ] -> [ ペンです ] (3単語)
```

## 2. RNN とトランスフォーマーブロックの関係

非常に重要な大前提として、**トランスフォーマーブロック（Transformer Block）の内部には、RNN は一切含まれていません。**

これらは系列データを処理するための、**歴史的にも仕組み的にも「対立・進化した」全く異なる2つのアーキテクチャ**です。

```text
【RNN（再帰型）】
  データを1語ずつ「ループ」で順番に処理する。
  (前回の記憶を引き継ぐため、前の計算が終わらないと次へ進めない)

【Transformer（アテンション型）】
  データを一括（並列）で処理する。
  (Attention機構を使い、すべての単語同士の関連度を一撃で計算する)
```

### トランスフォーマーブロックの内部構成
Transformerブロックを定義する際、内部に記述するのは以下のようなパーツです：
1.  **Multi-Head Attention (自己注意機構)**: 単語間の関連度を並列に計算する。
2.  **Feed Forward Network (FFN)**: 各単語の位置ごとに独立して処理を行う全結合層。
3.  **Layer Normalization (LayerNorm)**: データの分布を整えて学習を安定化させる。
4.  **Residual Connection (残差接続)**: 勾配消失を防ぐためのバイパス。

つまり、**「RNNを含むトランスフォーマーブロック」というものは存在せず**、モデルの設計者は「RNNで作るか」「Transformerで作るか」のどちらかを選択してネットワークを定義します。

（※ただし、歴史的に「RNNで解いていた系列モデリングのタスクを、アテンションだけで解くようにした」のがTransformerであるため、書籍の1章などでは両者の比較が必ず語られます）
