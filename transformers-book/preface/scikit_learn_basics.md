# scikit-learn のユースケースと LLM 連携

`scikit-learn`（サイキット・ラーン）は、ディープラーニング（PyTorchやTensorFlow）を除いた、**「伝統的（古典的）な機械学習アルゴリズム」を集約した Python の超定番ライブラリ**です。

---

## 1. scikit-learn の一般的な 5大ユースケース

| 分野 | 概要 | 具体的なユースケース | 代表的なアルゴリズム |
|---|---|---|---|
| **分類 (Classification)** | データをあらかじめ決められたクラス（カテゴリ）に分類する。 | スパムメール判定、病気の有無の診断、顧客の解約予測 | ロジスティック回帰、ランダムフォレスト、SVM |
| **回帰 (Regression)** | 連続する数値（量）を予測する。 | 家賃の予測、商品の売上予測、明日の気温予測 | 線形回帰、リッジ回帰、ランダムフォレスト回帰 |
| **クラスタリング** | 正解ラベルがないデータを、類似度に基づいてグループ分けする。 | 顧客の購買行動に応じたセグメンテーション、類似文書の自動整理 | K-Means, DBSCAN |
| **次元削減** | 高次元のデータを、情報損失を抑えつつ2〜3次元に圧縮する。 | 複雑なデータの可視化、データ軽量化によるノイズ除去 | PCA (主成分分析), t-SNE |
| **前処理と評価** | データのスケーリングや、予測精度の算出を行う。 | データの標準化（平均0, 分散1化）、訓練/テストデータの分割 | `StandardScaler`, `train_test_split`, F1-score等 |

---

## 2. 現代の LLM 開発における scikit-learn の使いどころ

「何でもLLM（ディープラーニング）で処理する」のは、計算コストや速度の観点から必ずしも最適ではありません。
現在、実務で非常によく使われるのが**「LLMの表現力」と「scikit-learnの軽さ・速さ」を組み合わせたハイブリッドアプローチ**です。

### 💡 「LLMによるベクトル化（Embedding）」 ＋ 「scikit-learn による高速分類」
1.  テキストデータを、LLM（Sentence-Transformersなど）に入力し、高次元の**「埋め込みベクトル（Embedding）」**として抽出します。
2.  そのベクトルを `scikit-learn` の単純な分類器（ロジスティック回帰やSVM）に入力して学習させます。

#### このアプローチのメリット
*   **超高速・低コスト**: 
    LLMを直接ファインチューニング（追加学習）するには巨大なGPUが必要で時間もかかりますが、`scikit-learn` の学習はCPUだけで**数ミリ秒〜数秒**で終わります。
*   **少ないデータで動く**: 
    LLMが既に言葉の意味（ベクトル）を理解しているため、`scikit-learn` 側はわずかな訓練データ（数十〜数百件）だけで高精度な分類器を構築できます。
